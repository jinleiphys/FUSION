# FRESCO output files and how to read observables

The main output (fort.6 or stdout) echoes the parsed input, then per-Jπ and per-energy summaries, then integrated cross sections and angular distributions. Detailed and plot-ready data go to numbered `fort.*` files written into the current working directory. **Always run in a scratch dir** so these do not overwrite anything.

## The integrated cross sections (in stdout)

At the end of each energy, look for the `CUMULATIVE` block:

```
0CUMULATIVE REACTION cross section  = <reaction σ>   <L> = ..  <L**2> = ..
0CUMULATIVE outgoing cross sections in partition <n> : <per-partition outgoing>
0Cumulative ABSORBTION by Imaginary Potentials = <absorption σ>
0CUMULATIVE OUTGOING cross section  = <total outgoing σ>
```

Interpretation:
- **REACTION** = total reaction cross section (flux lost from the elastic channel) = absorption + all outgoing (non-elastic) channels.
- **ABSORBTION by Imaginary Potentials** = flux removed by the optical `W`. For pure CDCC with real breakup couplings this is the fusion-like part; for the user's Line F flux-decomposition work this is exactly σ_abs.
- **OUTGOING** = flux into explicit outgoing channels (breakup bins, transfer, inelastic).
- Negative absorption with positive equal-magnitude outgoing (seen in `b8ex`) means the imaginary strength is small and the breakup flux is being counted as outgoing rather than absorbed; check the physics, it is not automatically a bug.

`scripts/check_xsec.py` parses exactly these lines.

## Common extraction recipes

- **Total reaction / absorption / breakup**: grep the CUMULATIVE block in stdout (see above).
- **Elastic angular distribution (ratio to Rutherford)**: fort.16 first dataset, or fort.201.
- **Breakup energy distribution** (dσ/dE per bin): fort.13, then `sumxen < fort.13 > x.xen` sums over partial waves per energy.
- **Total breakup angular distribution**: fort.16, then `sumbins < fort.16 > x.xsum` sums over bins.
- **Partial-wave convergence**: fort.56, check that reaction/nonelastic σ per J has died off well before jtmax.
- **Bound-state properties** (binding energy achieved, rms radius, ANC, D0): printed in stdout near the overlap setup, controlled by `ipc` in `&overlap`.

The auxiliary tools `sumbins`, `sumxen`, `sumxen` are separate small executables shipped with FRESCO (check `~/bin/`). If absent, the same sums can be scripted from fort.13/fort.16 columns.
