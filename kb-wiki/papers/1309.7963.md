# A graphics processor-based intranuclear cascade and evaporation simulation

> Monte Carlo simulations of the transport of protons in human tissue have been deployed on graphics processing units (GPUs) with impressive results. To provide a more complete treatment of non-elastic nuclear interactions in these simulations, we developed a fast intranuclear cascade-evaporation simulation for the GPU. This can be used to model non-elastic proton collisions on any therapeutically relevant nuclei at incident energies between 20 and 250 MeV. Predictions are in good agreement with Geant4.9.6p2. It takes approximately 2 s to calculate $1\times 10^6$ 200 MeV proton-$^{16}$O interactions on a NVIDIA GTX680 GPU. A speed-up factor of $\sim$20 relative to one Intel i7-3820 core processor thread was achieved.

## Key claim

The authors developed a fast intranuclear cascade-evaporation simulation for graphics processing units (GPUs) that models non-elastic proton collisions on therapeutically relevant nuclei at incident energies between 20 and 250 MeV, achieving predictions in good agreement with Geant4.9.6p2 while running approximately 20 times faster than a single Intel i7-3820 processor core thread.

## Method

The simulation implements a Bertini-type intranuclear cascade (INC) model where incident protons and nucleon-nucleon collision products are tracked through a nucleus modeled as a two-component fermion gas with a Fermi density distribution approximated by 15 constant-density shells, followed by a generalized evaporation model (GEM) for nuclear de-excitation. The code was implemented on a GPU using the CUDA framework with three kernels: random number initialization using CURAND, the INC simulation, and nuclear evaporation calculations; compile-time conditions allow the same program to run on CPU for debugging and timing comparisons.

## Key numbers

* Speed-up factor of approximately 20 relative to one Intel i7-3820 core processor thread for the GPU implementation
* Approximately 2 seconds to calculate 1×10^6 200 MeV proton-16O interactions on a NVIDIA GTX680 GPU
* For 2.6×10^6 200 MeV proton-16O interactions: 6.61 seconds on GPU with fast math option, 8.39 seconds on GPU without fast math, 172.74 seconds on CPU (same code), 492.42 seconds for Geant4 Bertini on CPU, and 2382.07 seconds for Geant4 Binary on CPU
* For 2.6×10^6 200 MeV proton-40Ca interactions: 9.77 seconds on GPU with fast math option, 15.56 seconds on GPU without fast math, 326.11 seconds on CPU (same code), 534.13 seconds for Geant4 Bertini on CPU, and 4620.92 seconds for Geant4 Binary on CPU
* For 2.6×10^6 70 MeV proton-16O interactions: 5.04 seconds on GPU with fast math option, 6.49 seconds on GPU without fast math, 138.21 seconds on CPU (same code), 556.57 seconds for Geant4 Bertini on CPU, and 2082.41 seconds for Geant4 Binary on CPU
* For 2.6×10^6 70 MeV proton-40Ca interactions: 7.19 seconds on GPU with fast math option, 11.83 seconds on GPU without fast math, 242.94 seconds on CPU (same code), 501.38 seconds for Geant4 Bertini on CPU, and 3220.15 seconds for Geant4 Binary on CPU
* Using CUDA math intrinsics further improved performance by a factor of 1.3 to 1.6

## Context

This work extends GPU-based proton transport Monte Carlo simulations, which previously made simplifying assumptions about non-elastic nuclear interactions: Kohno et al. did not directly simulate nuclear processes but used measured depth dose distributions, while Jia et al. followed Fippel and Soukup using a crude nuclear interaction model limited to oxygen. The authors note that existing GPU proton Monte Carlos cannot perform detailed investigations of secondary particles and residual nuclei, and that unsatisfactory results could occur in high-Z materials such as metallic implants. This simulation enables a fast and accurate GPU-based particle transport Monte Carlo for proton therapy dose calculations, with the overhead from non-elastic interactions estimated to be less than 10 seconds on a GTX680 for 1×10^7 proton trajectories, and also enables very fast calculation of positron emitter yields for range verification.
