# HPRMAT: A high-performance R-matrix solver with GPU acceleration for coupled-channel problems in nuclear physics

> I present HPRMAT, a high-performance solver library for the linear systems arising in R-matrix coupled-channel scattering calculations in nuclear physics. Designed as a drop-in replacement for the linear algebra routines in existing R-matrix codes, HPRMAT employs direct linear equation solving with optimized libraries instead of traditional matrix inversion, achieving significant performance improvements. The package provides four solver backends: (1) double-precision LU factorization, (2) mixed-precision arithmetic with iterative refinement, (3) a Woodbury formula approach exploiting the kinetic-coupling matrix structure, and (4) GPU acceleration. Benchmark calculations demonstrate that the GPU solver achieves up to 9$\times$ speedup over optimized CPU direct solvers, and 18$\times$ over legacy inversion-based codes, for large matrices ($N=25600$). The mixed-precision strategy is particularly effective on consumer GPUs (e.g., NVIDIA RTX 3090/4090), where single-precision throughput exceeds double-precision by a factor of 64:1; by performing factorization in single precision with iterative refinement, HPRMAT overcomes the poor FP64 performance of consumer hardware while maintaining double-precision accuracy. This makes large-scale CDCC and coupled-channel calculations accessible to researchers using standard desktop workstations, without requiring expensive data-center GPUs. CPU-only solvers provide 5--7$\times$ speedup through optimized libraries and algorithmic improvements. All solvers maintain physics accuracy with relative errors below $10^{-5}$ in cross-section calculations, validated against Descouvemont's reference code (Comput.\ Phys.\ Commun.\ 200, 199--219 (2016)). HPRMAT provides interfaces for Fortran, C, Python, and Julia.

## Key claim

HPRMAT achieves up to 9× speedup over optimized CPU direct solvers and 18× over legacy inversion-based codes for large R-matrix coupled-channel problems, but only for matrices of dimension N=25600 on consumer GPUs; the speedup is conditional on matrix size, with CPU solvers being faster for N < 400 due to GPU kernel launch and memory transfer overhead.

## Method

The paper implements four solver backends for the dense complex linear systems arising from Lagrange-Legendre basis discretization of the R-matrix internal region: (1) double-precision LU factorization via ZGESV from OpenBLAS; (2) mixed-precision arithmetic performing LU factorization in single precision (CGETRF) with iterative refinement in double precision; (3) a Woodbury formula approach exploiting the block structure where diagonal kinetic blocks are full matrices and off-diagonal coupling blocks are diagonal for local potentials; (4) GPU acceleration using NVIDIA cuSOLVER with mixed-precision on consumer GPUs. The solvers replace explicit matrix inversion with direct linear equation solving and are validated against Descouvemont's reference code (2016) using five standard test cases.

## Key numbers

- GPU solver achieves up to 9× speedup over optimized CPU direct solvers and 18× over legacy inversion-based codes for matrices of dimension N=25600 on an NVIDIA RTX 3090
- Single-precision throughput exceeds double-precision by a factor of 64:1 on consumer GPUs (RTX 3090: 35 TFLOPS FP32 vs. 0.56 TFLOPS FP64)
- All solvers maintain relative errors below 10^{-5} in cross-section calculations
- CPU-only solvers provide 5–7× speedup through optimized libraries and algorithmic improvements
- Mixed-precision solver achieves double-precision accuracy (~10^{-16}) with 1–2 iterative refinement iterations
- Woodbury-kinetic solver achieves 10–20% speedup over mixed-precision on CPU for large N, but with lower accuracy (10^{-6} vs. 10^{-16})
- GPU solver crossover point: for N < 400 CPU is faster; for N > 1000 GPU provides significant speedup
- Maximum matrix size tested on RTX 3090 (24 GB memory): N=25600; recommended maximum N≈21000 for 24 GB GPUs
- For GPUs with 8 GB memory (e.g., RTX 3070), maximum practical matrix size is N≈12000

## Context

This work extends Descouvemont's R-matrix package (Comput. Phys. Commun. 200, 199–219, 2016) by replacing its matrix inversion approach with direct linear solvers, and is the first publicly available nuclear-physics R-matrix code to exploit GPU acceleration and mixed-precision techniques. The UK PRMAT code (Noble, 2002) for atomic physics uses MPI parallelization and ScaLAPACK but does not employ GPU acceleration. HPRMAT enables large-scale CDCC and XCDCC calculations for halo nuclei such as 11Be and 6He on standard desktop workstations with consumer GPUs, where matrix dimensions of N=10000–30000 are routinely required for problems with 200–400 coupled channels.
