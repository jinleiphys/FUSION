# Particlization in fluid dynamical simulations of heavy-ion collisions: The iS3D module

> The iS3D particlization module simulates the emission of hadrons from heavy-ion collisions via Monte-Carlo sampling of the Cooper-Frye formula which converts fluid dynamical information into local phase-space distributions for hadrons. The code package includes multiple choices for the non-equilibrium correction to these distribution functions: the 14-moment approximation, first-order Chapman-Enskog expansion, and two types of modified equilibrium distributions. This makes it possible to explore to what extent heavy-ion experimental data are sensitive to different choices for $δf_n$, presently the main source of theoretical uncertainty in the particlization stage. We validate our particle sampler with a high degree of precision by generating several million hadron emission events from a longitudinally boost-invariant hypersurface and comparing the event-averaged particle spectra and space-time distributions to the Cooper-Frye formula.

## Key claim

The iS3D particlization module provides a validated Monte-Carlo sampler for the Cooper-Frye formula that includes four different choices for the non-equilibrium correction δf_n, enabling systematic exploration of how this choice, currently the main source of theoretical uncertainty in the particlization stage, propagates into uncertainties in extracted medium properties, but the sampler validation is performed only for a longitudinally boost-invariant central Pb-Pb collision at LHC energies.

## Method

The authors implemented a C++ code that samples hadron emission from a hydrodynamic particlization hypersurface using Monte-Carlo integration of the Cooper-Frye formula, with four options for the viscous correction δf_n: the 14-moment approximation, first-order Chapman-Enskog expansion, the Pratt-Torrieri-Bernhard (PTB) modified equilibrium distribution, and a new variant called PTM. The particle sampler was validated by generating several million hadron emission events from a (2+1)-dimensional longitudinally boost-invariant hypersurface produced by the GPU-VH hydrodynamic code for a central Pb-Pb collision, then comparing event-averaged sampled particle spectra and space-time distributions to the numerically evaluated continuous Cooper-Frye formula.

## Key numbers

* The typical running time to sample 1000 particlization events is about 94 seconds on a single-core Intel Xeon E5-2680 v4 CPU.
* Computing continuous momentum spectra on a p_T × φ_p = 100 × 48 grid takes about 56 seconds per particle on an Intel Xeon E5-2680 v4 multi-core processor with OpenMP and 2.1 seconds per particle on an Nvidia Tesla V100-PCIE GPU.
* The benchmark hypersurface from a longitudinally boost-invariant central Pb-Pb collision at LHC energies contains approximately 1.9 × 10^5 freezeout cells.
* The code requires about 1 GB of global memory to sample 1000 events from a 2+1d hypersurface.

## Context

This work extends the earlier iSS sampler from the iEBE-VISHNU simulation package (Shen, 2014) by adding support for 3-dimensionally expanding systems without longitudinal boost-invariance and implementing additional δf_n options beyond the standard 14-moment approximation. The code enables systematic studies of how the choice of δf_n parametrization affects Bayesian extractions of transport coefficients from experimental data, as previously performed by Bernhard (2018) using only the PTB distribution, and can provide input for hadronic afterburner codes such as UrQMD (Bass, 1998) and SMASH (Weil, 2016).
