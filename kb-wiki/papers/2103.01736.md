# Estimation of Impact Parameter and Transverse Spherocity in heavy-ion collisions at the LHC energies using Machine Learning

> Recently, machine learning (ML) techniques have led to a range of numerous developments in the field of nuclear and high-energy physics. In heavy-ion collisions, the impact parameter of a collision is one of the crucial observables which has a significant impact on the final state particle production. However, calculation of such a quantity is nearly impossible in experiments as the length scale ranges in the level of a few fermi. In this work, we implement the ML-based regression technique via Boosted Decision Tree (BDT) to obtain a prediction of impact parameter in Pb-Pb collisions at $\sqrt{s_{\rm NN}}$ = 5.02 TeV using A Multi-Phase Transport (AMPT) model. In addition, we predict an event shape observable, transverse spherocity in Pb-Pb collisions at $\sqrt{s_{\rm NN}}$ = 2.76 and 5.02 TeV using AMPT and PYTHIA8 based on Angantyr model. After a successful implementation in small collision systems, the use of transverse spherocity in heavy-ion collisions has potential to reveal new results from heavy-ion collisions where the production of a QGP medium is already established. We predict the centrality dependent spherocity distributions from the training of minimum bias simulated data and it was found that the predictions from BDT based ML technique match with true simulated data. In the absence of experimental measurements, we propose to implement Machine learning based regression technique to obtain transverse spherocity from the known final state observables in heavy-ion collisions.

## Key claim

The authors demonstrate that a Gradient Boosted Decision Tree (GBDT) machine learning regression model, trained on final-state charged-particle multiplicity and mean transverse momentum from AMPT and PYTHIA8 (Angantyr) simulations, can predict impact parameter and transverse spherocity in Pb-Pb collisions at LHC energies, but the predictions are validated only against simulated data, not experimental measurements.

## Method

The authors generated minimum-bias Pb-Pb collision events using the AMPT model (string melting version 2.26t7) at √sNN = 5.02 TeV and 2.76 TeV, and PYTHIA8 (Angantyr) at 5.02 TeV. They trained a Gradient Boosting Decision Tree regression model (scikit-learn GradientBoostingRegressor) using charged-particle multiplicity (⟨dNch/dη⟩), charged-particle multiplicity in the transverse region (⟨Nch^TS⟩), and mean transverse momentum (⟨pT⟩) as input variables, with impact parameter (b) and transverse spherocity (S0) as target variables. The model used 100 trees, maximum depth of 40, learning rate of 0.1, and least-squares loss function, trained on 60K events with testing on independent 16K event samples.

## Key numbers

- Mean absolute error for impact parameter prediction: Δb = 0.52 fm for Pb-Pb at √sNN = 5.02 TeV (testing data, 16K events)
- Mean absolute error for transverse spherocity prediction: ΔS0 = 0.055 for Pb-Pb at √sNN = 5.02 TeV (testing data, 16K events)
- Training data size dependence: Δb decreased from 0.71 fm (2K events) to 0.52 fm (60K events); ΔS0 decreased from 0.079 (2K events) to 0.055 (60K events)
- Transverse spherocity defined for events with at least 5 charged particles in |η| < 0.8 with pT > 0.15 GeV/c
- Correlation coefficient between impact parameter and charged-particle multiplicity shows significant anti-correlation (ρ negative, exact values in correlation matrix figures)

## Context

This work extends prior applications of machine learning for impact parameter determination in heavy-ion collisions (Bass 1993, Li 2020, David 1994, Bass 1996, Haddad 1996, De Sanctis 2009) and builds on the authors' own previous work (Mallick 2020) which first applied transverse spherocity in heavy-ion collisions using AMPT. The study enables future experimental measurements of transverse spherocity in heavy-ion collisions at LHC experiments (ALICE, CMS, ATLAS), where direct calculation is computationally challenging, by providing a regression method to estimate it from standard final-state observables.
