# FELIX-1.0: A finite element solver for the time dependent generator coordinate method with the Gaussian overlap approximation

> We describe the software package FELIX that solves the equations of the time-dependent generator coordinate method (TDGCM) in N-dimensions (N $\geq$ 1) under the Gaussian overlap approximation. The numerical resolution is based on the Galerkin finite element discretization of the collective space and the Crank-Nicolson scheme for time integration. The TDGCM solver is implemented entirely in C++. Several additional tools written in C++, Python or bash scripting language are also included for convenience. In this paper, the solver is tested with a series of benchmarks calculations. We also demonstrate the ability of our code to handle a realistic calculation of fission dynamics.

## Key claim

The paper presents FELIX-1.0, a finite element solver for the time-dependent generator coordinate method with the Gaussian overlap approximation (TDGCM+GOA), claiming that the finite element discretization enables more efficient scaling with the number of collective variables than previous finite difference approaches, though the code has only been tested in practice on 1, 2, and 3 dimensional meshes despite being designed for N dimensions (N ≥ 1).

## Method

The code discretizes the N-dimensional collective space using the Galerkin finite element method with simplex meshes (triangles for 2D, tetrahedra for 3D) and Lagrange polynomial bases of user-specified degree within each element. Time integration employs the Crank-Nicolson scheme, requiring inversion of sparse complex symmetric matrices at each time step via an iterative QMR algorithm with Jacobi preconditioning. An absorption band with a polynomial imaginary potential is placed along the domain boundary to simulate leakage of the wave packet outside the calculation region.

## Key numbers

- Memory usage for a 1D calculation with linear polynomials on a mesh with 600 nodes is approximately 3.3 MB
- Memory usage for a realistic 2D fission simulation with quadratic polynomials and 1.3 × 10^5 nodes reaches 1.5 GiB
- Two periods of a 1D harmonic oscillator (600 nodes, 800 time steps) are computed in a few seconds on one thread of an Intel Core i5 CPU
- A 2-dimensional realistic fission case (10^5 nodes, 10^5 time steps) requires roughly 10 hours on 10 threads of an Intel Xeon EP X5660 processor

## Context

This work extends prior TDGCM+GOA calculations by Berger (1984, 1991), Goutte (2005), and Younes (2012) that used finite difference discretization on regularly meshed hyper-cubes, which limited studies to 2-dimensional collective spaces. The finite element approach enables irregularly-spaced meshes and p-refinement techniques, allowing future calculations with the four or five collective variables that microscopic approaches by Moeller (2001), Younes (2009), Dubray (2012), and Schunck (2014) suggest are needed for fission dynamics.
