# SMASH as an event generator for heavy-ion collisions

> In this article we present an overview of the SMASH hadronic transport approach that is applied for non-equilibrium dynamics of hadrons in \heavyion\ collisions. We will give an overview about the ingredients of the approach and the applications for the dynamical description of \heavyion\ collisions and for calculations of fundamental properties of the hadron gas. The main emphasis of the article will be the infrastructure for sustainable software development that we have developed over the last 10 years including extensive unit tests and continuous integration. We will also provide one section about the performance of the code and how it can be analyzed and improved in the future.

## Key claim
SMASH is presented as a general-purpose hadronic transport event generator for heavy-ion collisions that solves the relativistic Boltzmann equation, but the paper's main emphasis is on the sustainable software development infrastructure built over the last 10 years rather than on new physics results. The authors state that the code has become roughly two times faster in most setups over the last couple of years, though they note that backward compatibility is not always easy to guarantee with new physics features.

## Method
SMASH solves the relativistic Boltzmann equation for all well-known hadronic species from the 2018 Particle Data Book, using a geometric collision criterion for binary collisions and a stochastic rates approach with test particles for multi-particle reactions. At low beam energies, reactions proceed via resonance excitation and decay with a mean-field potential, while at high energies strings are excited and fragment via Pythia. The code is written in C++17, built via CMake, and is modular so it can be used in parts or fully as a library, with coupling to hybrid approaches such as SMASH-vHLLE.

## Key numbers
- Over 150 hadronic species are included in the model, based on the particle data book from 2018.
- More than 100 test suites exist for unit, integration, and system tests.
- The code performance has roughly doubled in most setups over the last couple of years (e.g., between SMASH-2.2 and later releases).
- Au+Au collision snapshots are shown at E_kin = 0.8A GeV with impact parameter b = 3 fm and N_test = 20 test particles.
- The maximal arrow length in the velocity field corresponds to a velocity of 0.55 c.
- New public releases occur 1-2 times per year.

## Context
This work extends the long line of hadronic transport approaches developed over more than 30 years, including codes by Hartnack (1989), Bass (1998), Nara (1999), Lin (2004), Bratkovskaya (2011), Buss (2011), Pierog (2013), and Glassel (2021). The paper is part of the Transport Model Evaluation Project (TMEP, 2017-2022) which compared different frameworks. SMASH has been applied in hybrid approaches by Schafer (2021) and Gotz (2022), and by other groups including Nijs (2020), the JETSCAPE collaboration (2020), and Wu (2021). The code enables future work on multi-threading or parallel computing for CPU-intensive settings involving stochastic rates, many test particles, or parallel ensembles.
