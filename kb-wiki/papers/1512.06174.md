# PARTONS: PARtonic Tomography Of Nucleon Software. A computing framework for the phenomenology of Generalized Parton Distributions

> We describe the architecture and functionalities of a C++ software framework, coined PARTONS, dedicated to the phenomenology of Generalized Parton Distributions. These distributions describe the three-dimensional structure of hadrons in terms of quarks and gluons, and can be accessed in deeply exclusive lepto- or photo-production of mesons or photons. PARTONS provides a necessary bridge between models of Generalized Parton Distributions and experimental data collected in various exclusive production channels. We outline the specification of the PARTONS framework in terms of practical needs, physical content and numerical capacity. This framework will be useful for physicists - theorists or experimentalists - not only to develop new models, but also to interpret existing measurements and even design new experiments.

## Key claim

PARTONS is a C++ software framework that provides a standardized, modular architecture for the phenomenology of Generalized Parton Distributions (GPDs), bridging the gap between theoretical models and experimental data from exclusive production channels. The framework's layered and service-oriented design, derived from web-oriented technologies such as Java EE, is claimed to be original in scientific computing within nuclear and particle physics, though the paper presents the architecture rather than validating it against competing approaches.

## Method

The framework implements a layered architecture with seven parts: Module, Data, Result, Service, Database, Automation, and Visualization layers. Modules are encapsulated developments (GPD models, coefficient functions, cross sections) that receive standardized input objects (e.g., GPDKinematic containing x, xi, t, MuF2, MuR2) and return standardized output objects (e.g., GPDResult). A Registry singleton lists all available modules using self-registration via static variable initialization, while a Factory design pattern provides pre-configured copies of modules to users. Services combine multiple modules and data sets to provide high-level functions, and an Automation layer parses XML scenario files to execute computations without requiring users to write C++ code.

## Key numbers

- The GPDKinematic class stores five double variables: longitudinal momentum fraction x, skewness xi, squared four-momentum transfer t (in GeV^2), squared factorization scale MuF2 (in GeV^2), and squared renormalization scale MuR2 (in GeV^2)
- The example XML scenario specifies kinematic parameters: xB = 0.2, t = -0.1, Q2 = 2., and beam energy E = 6. (units not explicitly stated but GeV implied by context)
- The framework depends on the CLN library for one particular GPD module, with Qt as the only essential dependency for database connections, and SFML for thread handling

## Context

This work extends the phenomenology of GPDs, which describe the three-dimensional structure of hadrons in terms of quarks and gluons and are accessed in deeply exclusive lepto- or photo-production of mesons or photons. The framework is designed to support analysis of Jefferson Lab and COMPASS exclusive data over multiple years, and its modular structure allows addition of new models (e.g., the GK model of Goloskokov (2005-2009) or the VGG model of Vanderhaeghen (1998-2004)) without modifying existing code. The architecture could be extended to other quantities such as Transverse Momentum Distributions (TMDs), similar to the tmdlib project of Hautmann (2014).
