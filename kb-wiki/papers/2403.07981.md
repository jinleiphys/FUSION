# SPINAS: Spinor Amplitude Subroutines for Constructive Diagram Evaluations

> SPINAS is a C++ package created for the implementation and numerical computation of phase-space points of constructive amplitudes in particle physics. This package contains a suite of classes and methods for handling particles, propagators, spinor products, and processes. SPINAS is structured to offer straightforward usability while ensuring maximum efficiency. This is achieved through a design that emphasizes the storage and reuse of intermediate results within amplitude calculations for each phase-space point. We include a user guide describing how to use the components, a complete example of how to use SPINAS for a scattering amplitude, a discussion of the design and implementation useful for those wishing to contribute, and a discussion of our validation of this package, including both a validation of individual components of the package and a comparison of a complete set of Standard Model processes with Feynman diagrams.

## Key claim

SPINAS is a C++ package for numerical computation of phase-space points of constructive amplitudes in particle physics, designed to offer straightforward usability while ensuring maximum efficiency through storage and reuse of intermediate results; the paper presents the package's implementation and validation but notes that the constructive approach bypasses unphysical degrees of freedom and gauge parameters present in traditional Feynman diagram methods.

## Method

The package implements constructive amplitudes using spinor product classes (sproduct) that handle both massless helicity spinors and massive spin spinors, with particle and propagator classes for external and internal lines. The code stores and reuses intermediate results within amplitude calculations for each phase-space point, and validation was performed through unit tests of individual components plus comparison of a complete set of Standard Model 2-to-2 processes against Feynman diagram results.

## Key numbers

- The package requires CMake version 3.5 or higher for cross-platform compilation
- The C++ compiler must support the ISO/IEC 14882:2011 standard (C++11) or later
- Boost Unit Test framework version 1.71 or higher is required for module testing
- The ldouble type is defined as long double, typically occupying 16 bits on most modern architectures
- The v method for spinor products accepts integer arguments of only +1 (spin +1/2) or -1 (spin -1/2) for massive particle spin indices

## Context

This work extends the constructive amplitude formalism developed in Christensen (2018) and Christensen (2019), which found 3-point vertices in the constructive Standard Model and validated initial amplitudes against Feynman diagrams. The package is analogous to the HELAS Fortran routines (Helicity Amplitude Subroutines for Feynman Diagram Evaluation) that form the basis of modern matrix-element generators like MadGraph, but uses constructive amplitudes that avoid unphysical degrees of freedom. The package enables efficient numerical comparison of constructive amplitudes with Feynman diagrams and could potentially form the foundation for a new generation of matrix-element generators.
