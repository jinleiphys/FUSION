# An Interactive NeXTstep Interface to a Fortran Code for Solving Coupled Differential Equations

> This paper describes a user-friendly frontend to a Fortran program that integrates coupled nonlinear ordinary differential equations. The user interface is built using the NeXTstep Interface Builder, together with a public-domain graphical palette for displaying intermediate and final results. In running the code for a given set of equation parameters the user sees a plot of the solutions at each stage of the iterative process. In the case of a successful sequence of iterations, the initially discontinuous curves smooth out as the scale parameters of the solutions are adjusted to achieve a solution to the nonlinear equations. If the iterative process goes astray, as it often does for a poor choice of starting scale parameters, the user has the opportunity to stop and start over with a better choice, guided by the result of the previous run. The ease of interaction with the equations also allows the user to develop an intuition regarding their solutions and to explore the parameter space for the equations much more quickly.

## Key claim

This paper describes a user-friendly graphical frontend to a Fortran program that integrates coupled nonlinear ordinary differential equations, built using the NeXTSTEP Interface Builder and a public-domain plotting palette. The key innovation is overcoming the communication barrier between Objective-C (governing the NeXTSTEP interface) and Fortran by breaking the Fortran code into separate subroutines that compile as C function calls, though the approach is demonstrated for a specific physics problem (rho-stabilized skyrmions) rather than being a general-purpose solver.

## Method

The author converted an existing Fortran main program for solving coupled nonlinear ODEs (originally run as a batch process on a VAX) into several top-level subroutines corresponding to control buttons in the interface. The Fortran code was translated to C using the f2c compiler, allowing Objective-C methods in the NeXTSTEP application to call the Fortran subroutines as C functions, with data passed through intermediate files. The application uses a Runge-Kutta integration with a shooting method from the origin and from the asymptotic region to a matching radius, with a Newton-Raphson technique to iteratively adjust four scale parameters until the solutions become continuous and smooth at the match point.

## Key numbers

- The matching radius is typically set at 0.5 fm, with the backward integration starting from a large radius (e.g., 2 fm)
- The boundary value of F(0) is usually taken as pi, corresponding to baryon number B=1
- The asymptotic behavior of F(r) falls off as exp(-m_pi r)/r, and G(r) falls off as exp(-2 m_pi r) due to rho coupling to two pions
- The author reports that adapting the code to a different physics problem (pions and sigma-mesons) took approximately eight hours, with most time spent writing comment lines

## Context

This work extends the approach of using Numerical Recipes (Press et al., 1986) for Runge-Kutta integration of coupled ODEs, specifically for solving the two-boundary-value problem for rho-stabilized skyrmions as described by Igarashi et al. (1985). The paper demonstrates a practical method for creating interactive graphical interfaces for legacy Fortran scientific codes on NeXTSTEP workstations, enabling faster exploration of parameter space compared to batch-mode operation. The approach enables rapid reuse of the interface framework for different physics problems, requiring only changes to the equation-specific Fortran subroutines and minor Interface Builder modifications.
