# Self-learning Emulators and Eigenvector Continuation

> Emulators that can bypass computationally expensive scientific calculations with high accuracy and speed can enable new studies of fundamental science as well as more potential applications. In this work we discuss solving a system of constraint equations efficiently using a self-learning emulator. A self-learning emulator is an active learning protocol that can be used with any emulator that faithfully reproduces the exact solution at selected training points. The key ingredient is a fast estimate of the emulator error that becomes progressively more accurate as the emulator is improved, and the accuracy of the error estimate can be corrected using machine learning. We illustrate with three examples. The first uses cubic spline interpolation to find the solution of a transcendental equation with variable coefficients. The second example compares a spline emulator and a reduced basis method emulator to find solutions of a parameterized differential equation. The third example uses eigenvector continuation to find the eigenvectors and eigenvalues of a large Hamiltonian matrix that depends on several control parameters.

# Self-learning Emulators and Eigenvector Continuation

## Key claim
Self-learning emulation provides an active learning protocol that can achieve exponential or power-law error reduction using far fewer training points than random sampling, but only when the emulator faithfully reproduces exact solutions at training points and a fast error estimate proportional to the true error in the small-error limit is available.

## Method
The authors implement a greedy active learning algorithm that iteratively identifies the control parameter point with the largest estimated emulator error, adds that point to the training set, and retrains the emulator. For eigenvector continuation (EC), the fast error estimate uses the square root of the Hamiltonian variance evaluated with the approximate eigenvector, which is linearly proportional to the true eigenvector error in the small-error limit. The difference between estimated and actual error can be corrected using a Gaussian process trained at midpoints between training points.

## Key numbers
- Model 1 (transcendental equation): spline emulator error scales as approximately O(N^{-2.2}) with training points N; raw speedup factor s_raw ~ 10^5 (10^{-1} s per exact solution vs 10^{-6} s per emulator evaluation for 23 training points)
- Model 2 (differential equation): spline emulator error scales as O(N^{-1.88}) for N up to 300; reduced basis emulator error scales as O(e^{-2.66N}) for N above 10; spline raw speedup s_raw ~ 40 (7×10^{-2} s exact vs 1.7×10^{-3} s emulator for 23 points); RB raw speedup s_raw ~ 130 (7×10^{-2} s exact vs 5.5×10^{-4} s emulator for 13 points)
- Model 3 (eigenvector continuation, 4 particles on 4^3 lattice, 262,144-dimensional Hilbert space): EC error scales as O(e^{-0.27N}); raw speedup s_raw ~ 150 (1.95 s exact vs 0.013 s emulator for 41 training points); peak actual error after 40 iterations in 2D slice is ||Δv(c)|| = 2×10^{-5}; after 80 iterations over all six control parameters, peak error is 4×10^{-3}
- Model 3: constant offset A between log error estimate and log actual error is approximately -2.3

## Context
This work extends eigenvector continuation (Frame 2017, Frame 2019) and reduced basis methods (Quarteroni 2011, Field 2011) by introducing an active learning protocol that avoids the computational bottleneck of repeated exact eigenvector calculations. The self-learning approach differs qualitatively from gradient-descent machine learning methods, which cannot achieve arbitrarily high accuracy in polynomial time. The method enables multi-parameter studies of quantum many-body systems, such as particle clustering in four-particle systems with zero-range interactions, that would otherwise require prohibitive numbers of direct diagonalizations. The authors note that active learning for EC was first discussed by Eklind (2021), but their approach uses a fast variance-based error estimate rather than repeated exact calculations.
