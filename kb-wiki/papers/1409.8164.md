# The iEBE-VISHNU code package for relativistic heavy-ion collisions

> The iEBE-VISHNU code package performs event-by-event simulations for relativistic heavy-ion collisions using a hybrid approach based on (2+1)-dimensional viscous hydrodynamics coupled to a hadronic cascade model. We present the detailed model implementation, accompanied by some numerical code tests for the package. iEBE-VISHNU forms the core of a general theoretical framework for model-data comparisons through large scale Monte-Carlo simulations. A numerical interface between the hydrodynamically evolving medium and thermal photon radiation is also discussed. This interface is more generally designed for calculations of all kinds of rare probes that are coupled to the temperature and flow velocity evolution of the bulk medium, such as jet energy loss and heavy quark diffusion.

## Key claim

The iEBE-VISHNU code package provides a modular, open-source framework for event-by-event simulations of relativistic heavy-ion collisions using a hybrid (2+1)-dimensional viscous hydrodynamics plus hadronic cascade approach, with the specific shear viscosity \(\eta/s\) treated as a tunable free parameter to constrain QGP transport properties through model-data comparisons.

## Method

The package simulates heavy-ion collisions in multiple stages: initial fluctuating entropy density profiles are generated using Monte-Carlo Glauber (MC-Glauber) or Monte-Carlo Kharzeev-Levin-Nardi (MCKLN) models via the superMC code; these profiles are evolved with the (2+1)-dimensional viscous hydrodynamic code VISHNew; at a switching hypersurface, fluid cells are converted into particles using the iSS sampler; and the hadronic cascade code UrQMD v3.3 follows the particles microscopically until they stop interacting and decay. A Python driver script links all components for large-scale Monte-Carlo simulations, with final results stored in SQLite databases.

## Key numbers

- Running time for a single (2+1)-d hydrodynamics event (VISHNew) on a 400x400 grid: 120 seconds for p+p, 200 seconds for 0-5% p+Pb, and 690 seconds for 0-5% Pb+Pb at \(\sqrt{s_\mathrm{NN}} = 5.02\) TeV (tested on a 2.4 GHz Intel Core i5 CPU with GCC 4.9.2 and -O3 optimization).
- Running time for the hadron cascade UrQMD (1 event, \(|y| < 4\)): 0.03 seconds for p+p, 0.18 seconds for 0-5% p+Pb, and 150 seconds for 0-5% Pb+Pb at \(\sqrt{s_\mathrm{NN}} = 5.02\) TeV.
- Normalization factor \(\kappa_s\) for MC-Glauber model at Au+Au 200 A GeV ranges from 17.900 (for \(\eta/s = 0.08\)) to 14.290 (for \(\eta/s = 0.20\)).
- Normalization factor \(\kappa\) for MCKLN model at Au+Au 200 A GeV ranges from 5.692 (for \(\eta/s = 0.08\)) to 4.541 (for \(\eta/s = 0.20\)).
- The saturation scale parameterization uses \(\lambda = 0.2\) and \(\langle Q_s^2 (x = 0.01)\rangle \simeq 2.0\) GeV\(^2\) for central 200 A GeV Au+Au collisions.

## Context

This work extends the earlier VISHNU hybrid framework (Song 2010) by adding event-by-event capabilities, a modular Python driver, and a numerical interface for rare probes such as thermal photons. It provides a publicly available alternative to other hybrid codes like SONIC (Habich 2014), IPGlasma+MUSIC+UrQMD (Ryu 2015), and pure viscous hydrodynamic codes such as v-USPhydro, ECHO-QGP, and CLVisc. The package enables systematic model-data comparisons to constrain QGP transport coefficients, and its open-source design facilitates future extensions for jet energy loss and heavy quark diffusion studies.
