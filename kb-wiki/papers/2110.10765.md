# Accelerating quantum many-body configuration interaction with directives

> Many-Fermion Dynamics-nuclear, or MFDn, is a configuration interaction (CI) code for nuclear structure calculations. It is a platform-independent Fortran 90 code using a hybrid MPI+X programming model. For CPU platforms the application has a robust and optimized OpenMP implementation for shared memory parallelism. As part of the NESAP application readiness program for NERSC's latest Perlmutter system, MFDn has been updated to take advantage of accelerators. The current mainline GPU port is based on OpenACC. In this work we describe some of the key challenges of creating an efficient GPU implementation. Additionally, we compare the support of OpenMP and OpenACC on AMD and NVIDIA GPUs.

## Key claim

The paper demonstrates that directive-based GPU programming with OpenACC and OpenMP can effectively accelerate the nuclear configuration interaction code MFDn, but the performance and compiler support vary significantly across NVIDIA and AMD GPU architectures, with OpenACC generally outperforming OpenMP on both platforms for the kernels tested.

## Method

The authors ported three key computational motifs from the MFDn nuclear structure code to GPUs using OpenACC and OpenMP directive-based approaches: sparse matrix sparsity determination (counting nonzero matrix elements using bit representations and many-body state comparisons), parallel prefix sums for converting counts to offsets, and filling shared arrays with atomic operations. They benchmarked these kernels on NVIDIA V100, NVIDIA A100, and AMD MI100 GPUs using the NVIDIA Fortran compiler (version 21.7) and HPE/Cray Fortran compiler (version 12.0.1), comparing performance against Intel Skylake multi-core CPUs.

## Key numbers

- Matrix construction and observable evaluation each take about one-quarter to one-third of total MFDn runtime, while iterative eigensolving takes about one-third to one-half
- For the interacting state counting kernel with 8 particles and 128 single-particle states, the median density of nonzero elements was 6 × 10⁻⁶
- The parallel prefix sum implementation on A100 GPUs achieved over 500× speedup compared to a serial implementation for large arrays
- For the shared array filling kernel with m=512 and p=1, GPU implementations on MI100, A100, and V100 all outperformed Skylake CPUs (lower time is better, exact times shown in Figure 6)
- Test systems included NVIDIA V100 (16 GB per GPU, PCIe 3.0), NVIDIA A100 (40 GB per GPU, PCIe 4.0), and AMD MI100 (32 GB per GPU, PCIe 4.0)

## Context

This work extends the MFDn code's previous successful deployments on multi-core systems (Jaguar at OLCF, Mira at ALCF, Edison at NERSC) and many-core systems (Cori at NERSC, Theta at ALCF) as part of the NESAP application readiness program for NERSC's Perlmutter system. The GPU port of the LOBPCG eigensolver using OpenACC is described separately in Maris (2021). The paper identifies missing compiler support for array reductions in OpenACC 2.6 (Cray compiler) and OpenACC 2.7+ features (NVIDIA compiler), as well as limitations in OpenMP target offload for GPUs, enabling future work on portable directive-based GPU programming for nuclear structure calculations.
