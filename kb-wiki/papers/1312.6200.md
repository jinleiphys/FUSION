# Data mining the EXFOR database using network theory

> The EXFOR database contains the largest collection of experimental nuclear reaction data available as well as the data's bibliographic information and experimental details. We created an undirected graph from the EXFOR datasets with graph nodes representing single observables and graph links representing the various types of connections between these observables. This graph is an abstract representation of the connections in EXFOR, similar to graphs of social networks, authorship networks, etc. By analyzing this abstract graph, we are able to address very specific questions such as 1) what observables are being used as reference measurements by the experimental nuclear science community? 2) are these observables given the attention needed by various nuclear data evaluation projects? 3) are there classes of observables that are not connected to these reference measurements? In addressing these questions, we propose several (mostly cross section) observables that should be evaluated and made into reaction reference standards.

## Key claim

By constructing an undirected graph from the EXFOR database where nodes represent single observables and links represent connections between them, the authors identify which observables are used as reference measurements by the experimental nuclear science community and propose several (mostly cross section) observables that should be evaluated and made into reaction reference standards, though the graph is found to be disconnected, implying many reactions are not pinned to any monitor.

## Method

The authors used the x4i code to parse the REACTION and MONITOR fields from EXFOR entries and built an undirected graph with 87,925 nodes (each representing an elementary reaction/quantity) and 276,852 edges (representing monitor connections, mathematical relations, elemental links, and simultaneous evaluation links). They analyzed the graph using the networkx and graph-tool Python packages, studying cluster decomposition, degree distributions, and connectivity patterns to identify highly connected observables and disconnected components.

## Key numbers

- Total graph: 87,925 nodes and 276,852 edges
- Number of isolated nodes (absolute measurements): 23,196
- Number of clusters with 40 or more nodes: 7
- Average degree ⟨k⟩: 6.2975
- Degree variance ⟨k²⟩: 39.6584
- Probability two nodes are connected p: 7.162e-5
- Average cluster coefficient ⟨C⟩: 0.5958
- One main large cluster contains 35,415 nodes (less than half the total nodes)
- Second largest cluster contains 488 nodes and 963 edges (mostly photonuclear data)

## Context

This work extends the use of network theory to the EXFOR nuclear reaction database, building on prior efforts to compile and standardize nuclear data such as the ENDF/B Neutron Standards project (ongoing since 1968), the Atlas of Neutron Resonances by Mughabghab (5th edition), the IAEA medical radioisotope CRP (1995-2003), and the IRDFF dosimetry file (2012 version). The analysis enables identification of observables that are candidates for elevation to international reference standards, and reveals that many EXFOR entries are poorly coded, leading to a disconnected graph that limits the ability to trace connections between measurements and standards.
