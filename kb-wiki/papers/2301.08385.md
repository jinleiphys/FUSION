# Computational budget optimization for Bayesian parameter estimation in heavy ion collisions

> Bayesian parameter estimation provides a systematic approach to compare heavy ion collision models with measurements, leading to constraints on the properties of nuclear matter with proper accounting of experimental and theoretical uncertainties. Aside from statistical and systematic model uncertainties, interpolation uncertainties can also play a role in Bayesian inference, if the model's predictions can only be calculated at a limited set of model parameters. This uncertainty originates from using an emulator to interpolate the model's prediction across a continuous space of parameters. In this work, we study the trade-offs between the emulator (interpolation) and statistical uncertainties. We perform the analysis using spatial eccentricities from the T$_\mathrm{R}$ENTo model of initial conditions for nuclear collisions. Given a fixed computational budget, we study the optimal compromise between the number of parameter samples and the number of collisions simulated per parameter sample. For the observables and parameters used in the present study, we find that the best constraints are achieved when the number of parameter samples is slightly smaller than the number of collisions simulated per parameter sample.

## Key claim

For a fixed computational budget, the optimal constraints on model parameters in Bayesian inference for heavy ion collisions are achieved when the number of parameter samples (design points) is slightly smaller than the number of collisions simulated per parameter sample, specifically when the ratio of design points to events per design point is between 0.1 and 1, with an optimum near 0.25.

## Method

The authors used the T$_\mathrm{R}$ENTo model of initial conditions for heavy ion collisions to compute spatial eccentricities $\langle \varepsilon_n \rangle$ for n=2 to 5 in minimum bias Pb-Pb collisions at $\sqrt{s_{NN}}=2.76$ TeV. They performed closure tests by replacing experimental data with model calculations at a reference "truth" parameter set, then trained Gaussian process emulators on varying numbers of design points and ensemble sizes while keeping the total number of simulated events fixed. The posterior distribution from Bayesian inference was evaluated using two metrics: the value of the posterior at the true parameter value and the Akaike Information Criterion.

## Key numbers

- The optimal ratio of design points to events per design point ($N_d / M_{ev}$) was found to be approximately 0.25, corresponding to $N_d = 2^7$ design points when the total computational budget was $N_{tot} = 2^{16}$ events.
- The optimal number of design points was found to be between $N_d / M_{ev} = 0.1$ and 1, suggesting the optimal $N_d$ is of the order of $\sqrt{N_{tot}}$ or slightly smaller.
- The study used an inelastic nucleon-nucleon cross-section of 64 mb, corresponding approximately to $\sqrt{s_{NN}} = 2.76$ TeV collisions.
- The base case used $N_{tot} = N_d \times M_{ev} = 2^{16}$ total events and $M_{ev}^{truth} = 2^{16}$ events for the reference observables.

## Context

This work extends the methodology of Bayesian parameter estimation in heavy ion collisions, which has been applied in studies by Petersen (2010), Novak (2013), Bernhard (2015-2019), and the JETSCAPE collaboration (2020), among others. The study addresses the practical trade-off between emulator interpolation uncertainty and statistical uncertainty that arises when computational resources are limited, using the T$_\mathrm{R}$ENTo model (Moreland, 2014) as a proxy for heavy ion collision observables. The results enable more efficient allocation of computational resources in future Bayesian parameter estimation studies, particularly for models where the number of parameters and observables is large enough that emulator uncertainty becomes significant.
