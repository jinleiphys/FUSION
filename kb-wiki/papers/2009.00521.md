# Conception and software implementation of a nuclear data evaluation pipeline

> We discuss the design and software implementation of a nuclear data evaluation pipeline applied for a fully reproducible evaluation of neutron-induced cross sections of $^{56}$Fe above the resolved resonance region using the nuclear model code TALYS combined with relevant experimental data. The emphasis is on the mathematical and technical aspects of the pipeline and not on the evaluation of $^{56}$Fe, which is tentative. The mathematical building blocks combined and employed in the pipeline are discussed in detail. A unified representation of experimental data, systematic and statistical errors, model parameters and defects enables the application of the Generalized Least Squares (GLS) and its natural extension, the Levenberg-Marquardt (LM) algorithm, on a large collection of experimental data. The LM algorithm tailored to nuclear data evaluation accounts for the exact non-linear physics model to determine best estimates of nuclear quantities. Associated uncertainty information is derived from a Taylor expansion at the maximum of the posterior distribution. We also discuss the pipeline in terms of its IT (=information technology) building blocks, such as those to efficiently manage and retrieve experimental data of the EXFOR library and to distribute computations on a scientific cluster. Relying on the mathematical and IT building blocks, we elaborate on the sequence of steps in the pipeline to perform the evaluation, such as the retrieval of experimental data, the correction of experimental uncertainties using marginal likelihood optimization (MLO) and after a screening of thousand TALYS parameters -- including Gaussian process priors on energy dependent parameters -- the fitting of about 150 parameters using the LM algorithm. The code of the pipeline including a manual and a Dockerfile for a simplified installation is available at www.nucleardata.com.

## Key claim

The paper presents the design and software implementation of a fully reproducible nuclear data evaluation pipeline for neutron-induced cross sections of $^{56}$Fe above the resolved resonance region, but the authors explicitly state that the emphasis is on the mathematical and technical aspects of the pipeline and not on the evaluation of $^{56}$Fe itself, which they describe as "tentative."

## Method

The pipeline combines the nuclear model code TALYS with experimental data from the EXFOR library, using a unified representation of experimental data, systematic and statistical errors, model parameters, and defects to enable application of Generalized Least Squares (GLS) and the Levenberg-Marquardt (LM) algorithm. The LM algorithm accounts for the exact non-linear physics model to determine best estimates of nuclear quantities, with uncertainty information derived from a Taylor expansion at the maximum of the posterior distribution. The pipeline includes steps for retrieval of experimental data, correction of experimental uncertainties using marginal likelihood optimization (MLO), screening of approximately 1000 TALYS parameters including Gaussian process priors on energy-dependent parameters, and fitting of about 150 parameters using the LM algorithm.

## Key numbers

* The pipeline screens approximately 1000 TALYS parameters before fitting
* The fitting step uses about 150 parameters with the LM algorithm
* The evaluation is applied to neutron-induced cross sections of $^{56}$Fe above the resolved resonance region
* The pipeline code is available at www.nucleardata.com with a Dockerfile for simplified installation

## Context

This work extends the application of Bayesian statistical methods to nuclear data evaluation by providing a complete, reproducible software pipeline that integrates the TALYS nuclear model code with experimental data from the EXFOR library. The approach builds on prior work in nuclear data evaluation using Generalized Least Squares methods and extends it with the Levenberg-Marquardt algorithm for non-linear models, Gaussian process priors for energy-dependent parameters, and marginal likelihood optimization for uncertainty correction. The pipeline enables future reproducible evaluations of other isotopes and reaction channels, and the authors provide the complete software infrastructure including version-controlled packages with cryptographic hash identifiers for long-term reproducibility.
