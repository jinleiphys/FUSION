# pynucastro 2.1: an update on the development of a python library for nuclear astrophysics

> pynucastro is a python library that provides visualization and analyze techniques to classify, construct, and evaluate nuclear reaction rates and networks. It provides tools that allow users to determine the importance of each rate in the network, based on a specified list of thermodynamic properties. Additionally, pynucastro can output a network in C++ or python for use in simulation codes, include the AMReX-Astrophysics simulation suite. We describe the changes in pynucastro since the last major release, including new capabilities that allow users to generate reduced networks and thermodynamic tables for conditions in nuclear statistical equilibrium.

# pynucastro 2.1: an update on the development of a python library for nuclear astrophysics

## Key claim

pynucastro 2.1 provides new capabilities for generating reduced networks and thermodynamic tables for nuclear statistical equilibrium conditions, but these features are driven specifically by the developers' own science applications (convective burning in massive stars, core-collapse supernovae) and the weak-rate library expansion is limited to nuclei with A=45-65 from specific tabulated sources, not a comprehensive coverage of all relevant nuclei.

## Method

The authors extended the pynucastro library by adding weak-rate tabular data for nuclei A=45-65 from Langanke et al. (1999, 2000, 2001), reformatted with a bilinear interpolation scheme over log10(rho*Ye) and log10(T) for both Python and C++ network classes. They implemented a Directed Relation Graph with Error Propagation (DRGEP) modified reduction method that uses a modified Dijkstra's algorithm to compute path weights from target nuclei and removes species with weights below user-specified tolerances. They also added a SimpleCxxNetwork class for pure C++ output, a custom rate class framework, a Julia integration interface via DifferentialEquations.jl, and an NSENetwork class for constructing NSE tables with binning and weak-interaction evolution capabilities.

## Key numbers

* From a pool of 6758 rates with 584 nuclei (all REACLIB rates up to Ni-56), the DRGEP reduction method with 25 target nuclei and tolerances of 0.3-0.5 produced a reduced network of 607 rates with 63 nuclei
* The weak-rate library was expanded from A=17-28 (pynucastro 2.0) to include A=45-65 from Langanke et al. (1999, 2000, 2001)
* The DRGEP reduction example used density range (1e2, 1e6) g/cm^3 and temperature range (1.0e7, 3.0e8) K
* The Julia integration example used a CNO network with rho=150 g/cm^3 and T=1.5e7 K, integrated over tspan=(0.0, 1.e20) with absolute and relative tolerances of 1e-9

## Context

This work extends the pynucastro library described in the 2.0 milestone (Clark et al., 2023) and complements other reaction network libraries like SkyNet (Lippuner & Roberts, 2017) and WinNet (Reichert et al., 2023), but with a focus on interactive exploration and export to multidimensional simulation codes such as Castro and MAESTROeX in the AMReX-Astrophysics suite. The DRGEP reduction method is adapted from combustion chemistry approaches by Pepiot-Desjardins & Pitsch (2008) and Niemeyer & Sung (2011). The NSE table construction capability follows the approach of Seitenzahl et al. (2009). Future work includes adding a Helmholtz equation of state (Timmes & Swesty, 2000) for self-heating burn in Python and enabling machine learning applications through the Python interface.
