# Performing Bayesian analyses with AZURE2 using BRICK: an application to the ${}^7$Be system

> Phenomenological $R$-matrix has been a standard framework for the evaluation of resolved resonance cross section data in nuclear physics for many years. It is a powerful method for comparing different types of experimental nuclear data and combining the results of many different experimental measurements in order to gain a better estimation of the true underlying cross sections. Yet a practical challenge has always been the estimation of the uncertainty on both the cross sections at the energies of interest and the fit parameters, which can take the form of standard level parameters. Frequentist ($χ^2$-based) estimation has been the norm. In this work, a Markov Chain Monte Carlo sampler, \texttt{emcee}, has been implemented for the $R$-matrix code \texttt{AZURE2}, creating the Bayesian $R$-matrix Inference Code Kit (\texttt{BRICK}). Bayesian uncertainty estimation has then been carried out for a simultaneous $R$-matrix fit of the $^3$He$(α,γ)^7$Be and $^3$He$(α,α)^3$He reactions in order to gain further insight into the fitting of capture and scattering data. Both data sets constrain the values of the bound state $α$-particle asymptotic normalization coefficients in $^7$Be. The analysis highlights the need for low-energy scattering data with well-documented uncertainty information and shows how misleading results can be obtained in its absence.

## Key claim
Bayesian uncertainty quantification using the Markov Chain Monte Carlo sampler `emcee` has been implemented for the R-matrix code `AZURE2` via the new `BRICK` interface, but the analysis of the 7Be system shows that the results are highly sensitive to the choice of low-energy scattering data; misleading results can be obtained when scattering data with poorly documented uncertainties are used.

## Method
The authors coupled the R-matrix code `AZURE2` with the MCMC sampler `emcee` through a Python interface called `BRICK` (Bayesian R-matrix Inference Code Kit). They performed a simultaneous Bayesian R-matrix fit of the 3He(α,γ)7Be capture reaction and the 3He(α,α)3He elastic scattering reaction, sampling 16 R-matrix parameters plus normalization factors and energy shifts for each data set. The analysis used uniform priors on R-matrix parameters and Gaussian priors on normalization factors, with the likelihood constructed as a product of Gaussian likelihoods for each data point.

## Key numbers
* The R-matrix model used nine levels, three particle pairs, and 16 R-matrix fit parameters.
* The Barnard (1964) scattering data covered 2.4 ≤ E[3He, lab] ≤ 5.7 MeV (1.4 ≤ E_c.m. ≤ 3.3 MeV) with 646 data points at eight angles; systematic uncertainty was estimated at 5%, point-to-point uncertainties at about 3%, and energy uncertainties of 20-40 keV.
* The new SONIK scattering data covered nine energies from E_c.m. = 0.38 to 3.13 MeV with normalization uncertainties ranging from 4.1% to 9.8% per beam energy.
* The capture data sets covered E_c.m. = 94 to 3130 keV, with common-mode errors ranging from 3% (Seattle data) to 8% (Notre Dame data).
* The prior on the energy shift for SONIK data had a standard deviation of 3 keV; for the Barnard data, the prior had a standard deviation of 40 keV.

## Context
This work extends the R-matrix analysis of the 3He(α,γ)7Be and 3He(α,α)3He reactions previously performed by deBoer (2014) using χ² minimization with Monte Carlo simulation of uncertainties. It provides a Bayesian alternative to the frequentist uncertainty estimation that has been the norm in phenomenological R-matrix analyses since Lane and Thomas (1958). The `BRICK` code enables future Bayesian R-matrix analyses of other nuclear systems, and the results highlight the need for low-energy scattering data with well-documented uncertainty information, such as the new SONIK measurements by Paneru, to obtain reliable cross section extrapolations for nuclear astrophysics.
