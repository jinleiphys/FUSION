# Centroids of nuclear shell-model Hamiltonians, with optimization of energy-based truncation schemes

> The configuration-interaction shell model is an effective and widely-used approach to the nuclear many-body problem, whose main drawback is the exponential growth of the basis dimension. An useful way to character nuclear shell-model Hamiltonians is through traces, including traces in subspaces defined by orbital occupations. Such traces, or energy centroids, can be easily and efficiently computed through the monopole components of the nuclear interaction, that is, terms that go like $n_a n_b$ where $n_a$ is the occupation of the single-particle orbital labeled by $a$. These calculations can be carried out very quickly for both empirical (valence space) and no-core shell model spaces and interactions. In fact, they can be carried out so fast, one can use this to optimize an efficient, if approximate, many-body truncation scheme used in available nuclear shell-model codes such as BIGSTICK. To carry out both the traces and the optimization, we present the TRACER code, written in Fortran90 and described and available here. We give example results as well as discuss performance.

## Key claim

Configuration centroids (average energies) of nuclear shell-model Hamiltonians can be computed extremely efficiently using only the monopole components of the interaction, and this speed enables optimization of integer orbital weights for an approximate energy-based truncation scheme that outperforms standard \(N_{\text{max}}\) truncation in capturing low-energy configurations.

## Method

The authors compute configuration centroids using the monopole Hamiltonian, which depends only on orbital occupation numbers \(n_a\) and the monopole potential \(U(ab)\) derived from two-body matrix elements. They then define a fitness function \(\Phi\) that penalizes configurations above an energy cutoff \(E_{\text{cut}}\) that are included in the truncated space and rewards configurations below the cutoff that are excluded, weighted by subspace dimension \(d_\alpha\). A Monte Carlo search over integer orbital weights \(\vec{w}\) and maximum total weight \(W_{\text{max}}\) minimizes this fitness function to produce an optimized truncation scheme.

## Key numbers

- For \(^{60}\text{Fe}\) in the \(1p0f\) shell with GX1A interaction: 3600 combined configurations, 1,502,182,564 total states, centroid energy \(-173.9236\) MeV, lowest configuration energy \(-219.141\) MeV, width of centroids \(9.421\) MeV.
- For \(^{40}\text{Ar}\) in the \(sd\)-\(pf\) cross-shell space: full space dimension over \(9 \times 10^{14}\); \(N_{\text{max}}=0\) dimension 1566; \(N_{\text{max}}=2\) dimension 9 million; \(N_{\text{max}}=4\) dimension 4.6 billion; optimized \(W=3\) truncation dimension 604 million; optimized \(W=5\) truncation dimension 35 billion.
- Optimization for \(^{40}\text{Ar}\) with cutoff \(-200\) MeV: initial cost 8,151,709,768,804; final cost 5,295,343,403,008; optimal weights [1, 2, 1, 3, 2, 3, 2]; optimal maximum excitation 5.

## Context

This work extends spectral distribution theory (French 1967, Wong 1986) by focusing on first moments (centroids) of occupation-configuration subspaces, building on earlier suggestions for centroid-based truncation (PhysRevC.50.R2274). The TRACER code interfaces with the BIGSTICK shell-model code (Johnson 2018) and provides a practical alternative to group-theoretical truncations such as seniority (PhysRevC.59.2033), SU(3) (Dytrych 2016), or symplectic Sp(3,R) (Launey 2016), which are computationally expensive for partitioning. The optimized truncation enables larger shell-model calculations by capturing low-energy configurations missed by standard \(N_{\text{max}}\) truncation, though it does not exactly separate spurious center-of-mass motion.
