# Nuclear shell-model code for massive parallel computation, "KSHELL"

> A new code for nuclear shell-model calculations, "KSHELL", is developed. It aims at carrying out both massively parallel computation and single-node computation in the same manner. We solve the Schrödinger's equation in the $M$-scheme shell-model model space, utilizing Thick-Restart Lanczos method. During the Lanczos iteration, the whole Hamiltonian matrix elements are generated "on-the-fly" in every matrix-vector multiplication. The vectors of the Lanczos method are distributed and stored on memory of each parallel node. We report that the newly developed code has high parallel efficiency on FX10 supercomputer and a PC with multi-cores.

## Key claim

The KSHELL code achieves high parallel efficiency for nuclear shell-model calculations on both the FX10 supercomputer and multi-core PCs, but this performance is demonstrated only for a specific test case (⁵⁶Ni in the pf-shell) and the code's applicability to even larger dimensions (O(10¹⁰) or more) is stated as hopeful rather than proven.

## Method

The code solves the Schrödinger equation in the M-scheme shell-model space using the Thick-Restart Lanczos method, generating all Hamiltonian matrix elements "on-the-fly" during each matrix-vector multiplication. The M-scheme basis is split into subspaces called "partitions" or "sectors" defined by occupation numbers of each single-particle orbit and the z-component of angular momentum of protons, enabling parallel distribution of Lanczos vectors across MPI nodes. The code is written in Fortran 95 with ISO TR15581 extension and includes a dialogue-type user interface in Python 2.

## Key numbers

- Ground-state energy of ⁵⁶Ni in the pf-shell computed using 8192 cores on the FX10 supercomputer in 145 seconds, corresponding to an eigenvalue problem of a 1,087,455,228-dimensional matrix.
- Strong scaling demonstrated up to 8192 cores on the FX10 supercomputer.
- Two Lanczos vectors for a 10¹⁰-dimensional M-scheme space require approximately 80 GB of memory.
- The code copes with O(10¹⁰) M-scheme dimension and, hopefully, larger dimension.

## Context

This work extends the lineage of M-scheme shell-model codes such as Antoine (Caurier, 2005), MSHELL64 (Utsuno, 2010), and BIGSTICK (Johnson, 2013), but is the first publicly available code applicable to MPI parallel computation. The code enables large-scale shell-model calculations on massively parallel supercomputers, overcoming the memory bottleneck of storing Lanczos vectors by distributing them across nodes, and can compute quadrupole and magnetic moments as well as E2, M1, and E1 transition probabilities without additional manipulation. Three-body forces are outside the code's scope.
