# Computing the coefficients of transformations between oscillator states

> A program is created to compute recursively the Moshinsky brackets. It is very fast and provides highly accurate results. In the case of the double-precision computations with a single-processor consumer notebook, the computing time per bracket at any not small oscillator excitations is on the scale of 10^{-8} s and the accuracy is very good for the total number of quanta up to 80. The program is easy to handle.

## Key claim

The paper presents a recursive program for computing Moshinsky brackets (Talmi-Mirnov-Smirnov coefficients) that is very fast and highly accurate for total oscillator quanta up to 80 in double precision on a consumer notebook, but the abstract's claim of "very good accuracy" is qualified by the finding that at (N_q)_max values not much higher than 84, accuracy becomes insufficient in general.

## Method

The program implements recurrence relations from Efros (1973) that compute modified oscillator brackets using n_1-1 → n_1 and n_2-1 → n_2 recursion, starting from initial brackets with n_1 = n_2 = 0 given by an explicit expression involving 3j-symbols and sums over binomial coefficients. The recurrence coefficients ψ_L^± depend only on two quantum numbers l_1' and l_2', allowing precomputation for speed. Two subroutine versions are provided: ALLOSBRAC computes all brackets for given N_q, L ranges, and parity into a single array; OSBRAC computes subsets for given l_1, l_2, L values.

## Key numbers

- Computing time per bracket at any not small oscillator excitations is on the scale of 10^{-8} s in double precision on a single-processor consumer notebook
- Accuracy is very good for total number of quanta up to 80
- Maximum allowed (N_q)_max ≤ 84 in double precision to avoid factorial overflow/underflow
- At (N_q)_max = 8, the sum δ of Eq. (del) equals 7.9·10^{-13}; at (N_q)_max = 28, δ = 1.5·10^{-9}
- Running time t_1 for ALLOSBRAC at (N_q)_max = 8 is 1.9·10^{-4} s; at (N_q)_max = 28 is 0.28 s
- Running time t_2 for OSBRAC at (N_q)_max = 8 is 1.3·10^{-3} s; at (N_q)_max = 28 is 22 s
- At (N_q)_max = 40 and L_min = L_max = L, the BRAC array size is about 1 GB or less at any L
- At (N_q)_max = L_max = 20 and L_min = 0, the BRAC array size is about 12 GB
- For (N_q)_max = 21, L = 11, running time t_Σ = 1.7·10^{-3} s for all brackets with N_q ≤ (N_q)_max
- For (N_q)_max = 28, L = 10, running time t_Σ = 1.4·10^{-2} s

## Context

This work extends the recursive approach of Efros (1973) for computing Moshinsky brackets, providing a practical implementation that outperforms explicit-expression-based programs such as those of Gusev and Kurbatov (2018) and Litvinova (2019) in both speed and accuracy at high oscillator excitations. The program enables large-scale shell-model calculations in nuclear physics that require many oscillator brackets, particularly for computing matrix elements of nucleon-nucleon interactions in oscillator bases and for constructing properly symmetrized basis states in three-particle problems.
