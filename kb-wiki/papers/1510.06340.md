# A MUSTA-FORCE algorithm for solving partial differential equations of relativistic hydrodynamics

> Understanding event-by-event correlations and fluctuations is crucial for the comprehension of the dynamics of heavy ion collisions. Relativistic hydrodynamics is an elegant tool for modeling these phenomena; however, such simulations are time-consuming, and conventional CPU calculations are not suitable for event-by-event calculations. This work presents a feasibility study of a new hydrodynamic code that employs graphics processing units together with a general MUSTA-FORCE algorithm (Multi-Stage Riemann Algorithm - First Order Centered scheme) to deliver a high-performance yet universal tool for event-by-event hydrodynamic simulations. We also investigate the performance of selected slope limiters that reduce the amount of numeric oscillations and diffusion in the presence of strong discontinuities and shock waves. The numerical results are compared to the exact solutions to assess the code's accuracy.

# A MUSTA-FORCE algorithm for solving partial differential equations of relativistic hydrodynamics

## Key claim

The MUSTA-FORCE algorithm implemented on a GPU can deliver over 200 times speedup compared to sequential CPU implementations, making event-by-event relativistic hydrodynamic simulations feasible on a few modern workstations, but the method produces numerical oscillations that require careful selection of slope limiters, with the Minbee limiter showing the best numerical properties across all test cases.

## Method

The authors implemented a hybrid MUlti-STAge (MUSTA) approach using a First ORder CEntered (FORCE) flux in a predictor-corrector loop that solves the Riemann problem numerically without a priori wave information, extended to second-order accuracy in space and time via the MUSCL-Hancock scheme. They tested four slope limiters (Minbee, Superbee, van Albada, and van Leer) to reduce numerical oscillations and diffusion at discontinuities, and implemented the algorithm on NVIDIA GPUs using the CUDA framework with surface memory to store simulation data as 5 single-precision floating-point numbers per lattice cell.

## Key numbers

- GPU implementation achieved over 200 times speedup compared to a sequential CPU implementation (based on prior work cited as FedCSIS2013)
- Maximum grid that fits within surface memory limitations is 240^3 cells
- Sod shock tube simulation used grid size of 500 cells with grid spacing 0.02 and time step 0.005; left pressure p_L = 13 1/3, right pressure p_R = 0, left charge density n_L = 10, right charge density n_R = 1
- Hubble-like expansion simulation used grid size of 120^3 cells with grid spacing 0.1 and time step 0.03; energy density e_0 = 1, speed of sound squared c_s^2 = 1/3, proper time tau_0 = 4, initial time t_0 = 2
- Ellipsoidal flow simulation used grid size of 120^3 cells with grid spacing 0.1 and time step 0.02; constants C_e = 2, C_n = 0.75, b_e = 1, b_n = 1, T_1 = 0.4, T_2 = 0.6, T_3 = 0.8
- Four MUSTA stages guaranteed stability and good numerical accuracy in acceptable calculation time on GPU
- Third order accurate Runge-Kutta method was used for time integration

## Context

This work extends prior GPU-based hydrodynamic implementations such as the SHASTA algorithm (Gerhard 2013) which showed up to two orders of magnitude improvement, and the authors' own previous MUSTA-FORCE implementation (FedCSIS2013) that proved over 200 times faster than sequential CPU. The paper addresses the need for (3+1)-dimensional high-resolution simulations with fluctuating initial conditions, which are computationally prohibitive on CPUs, and enables event-by-event analyses using models like Monte-Carlo Glauber, CGC, SPheRIO, NeXus, NeXSPheRIO (Derradi de Souza 2011), EPOS, and UrQMD for initial conditions. The work aims to combine bulk physics described by hydrodynamic models with jet physics to study how energy deposited by jets affects soft particle behavior.
