# How to explain ENDF-6 to computers: A formal ENDF format description language

> The ENDF-6 format, widely used worldwide for storing and disseminating nuclear data, is managed by the Cross Sections Evaluation Working Group (CSEWG) and fully documented in the ENDF-6 formats manual. This manual employs a combination of formal and natural language, introducing the possibility of ambiguity in certain parts of the format specification. To eliminate the possibility of ambiguity, this contribution proposes a generalization of the formal language used in the ENDF-6 formats manual, which is precisely defined in extended Backus-Naur form (EBNF). This formalization also offers several other advantages, notably reducing the complexity of creating and updating parsing and validation programs for ENDF-6 formatted files. Moreover, the availability of a formal description enables the automatic mapping of the low-level ENDF-6 representation, provided by a sequence of numbers, to a more human-friendly hierarchical representation with variable names. Demonstrating these advantages, almost the entire ENDF-6 format description has been translated into the proposed formal language, accompanied by a reference implementation of a parser/translator. This implementation leverages the formal format specification to enable reading, writing, and validating ENDF-6 formatted files.

## Key claim

The paper proposes a formal ENDF-6 format description language defined in extended Backus-Naur form (EBNF) to eliminate ambiguity in the existing manual, which combines formal and natural language; the authors claim that almost the entire ENDF-6 format description has been translated into this formal language, accompanied by a reference parser implementation.

## Method

The authors defined a formal grammar using extended Backus-Naur form (EBNF) notation compatible with the Lark Python parsing toolkit, specifying production rules for ENDF-6 record types (HEAD, CONT, TAB1, TAB2, LIST, TEXT, DIR, INTG, SEND), control structures (FOR loops, IF clauses), sections for variable scoping, and arithmetic expressions. They then implemented a Python package called endf-parserpy that uses this formal grammar to parse ENDF-6 files into nested Python dictionaries, enabling reading, writing, validation, and translation to formats such as JSON.

## Key numbers

- The formal grammar defines production rules for 10 ENDF-6 record types: list_line, head_or_cont_line, tab1_line, tab2_line, text_line, dir_line, intg_line, send_line, dummy_line, and stop_line
- The reference implementation endf-parserpy can translate ENDF-6 files to JSON using a three-step process: parsefile("input.endf"), json.dump(endf_dict, f, indent=2)
- The example MF=13 (photon production cross section) recipe uses a conditional block "if NK>1" and a FOR loop "for k=1 to NK" with NK subsections
- The parser can detect inconsistencies such as a value mismatch where "Expected 39.6191 in the ENDF file but got 39.6564" for the variable AWR

## Context

This work extends the ENDF-6 format manual (Brown, editor, 2023) by formalizing its description language to remove ambiguity, complementing other nuclear data tools including ENDFtk (Conlin, 2017), FUDGE (Mattoon, 2023), DeCE (Kawano, 2019), SANDY (Fiorito, 2019), and endf-parser (Romano, 2023). The formalization enables automatic mapping between the physical data model (ENDF-6 file structure) and logical data model (symbolic variable names), and facilitates translation to JSON, XML, and HDF5 formats, which the authors argue will accelerate production and consumption of evaluated nuclear data.
