# tParton: Implementation of next-to-leading order evolution of transversity parton distribution functions

> We provide code to solve the Dokshitzer-Gribov-Lipatov-Altarelli-Parisi (DGLAP) evolution equations for the nucleon transversity parton distribution functions (PDFs), which encode nucleon transverse spin structure. Though codes are widely available for the evolution of unpolarized and polarized PDFs, there are few codes publicly available for the transversity PDF. Here, we present Python code which implements two methods of solving the leading order (LO) and next-to-leading order (NLO) approximations of the DGLAP equations for the transversity PDF, and we highlight the theoretical differences between the two.

# tParton: Implementation of next-to-leading order evolution of transversity parton distribution functions

## Key claim

The authors provide Python and Mathematica code that implements two methods for solving the DGLAP evolution equations for the nucleon transversity PDF at LO and NLO, but note that a discrepancy exists between the two methods (the Hirai t-step integration method and the Vogelsang Mellin moment method) due to instability in the numerical Mellin inversion and different choices of the strong coupling constant approximation.

## Method

The DGLAP evolution equation for the transversity PDF is solved using two approaches: (1) the Hirai method, which integrates the equation in t-space using the Euler method with Simpson's rule for convolution integrals on a discretized x-grid; and (2) the Vogelsang method, which computes Mellin moments of the splitting functions and initial PDF, applies the evolution factor in Mellin space, then performs an inverse Mellin transform using Cohen's method for fast Laplace inversion. The authors verify the Mellin moments of the LO and NLO splitting functions numerically in Mathematica against the analytic expressions given by Vogelsang.

## Key numbers

- Relative error between numerical moments and analytic moments of the splitting functions was at most 0.15% for Nc=3 and Nf=5
- Evolution performed from Q²=4 GeV² to Q²=200 GeV² using the GS-A distribution as input
- For the Vogelsang method, degree of approximation d_approx >= 5 is needed for accurate results, with no additional benefit beyond d_approx=7
- To achieve 1% relative error with the Vogelsang method, recommend 5 <= d_approx <= 7 and N_x' >= 200
- To achieve 0.1% relative error with the Hirai method, recommend N_x >= 200 and N_t >= 100
- Both methods at these settings took under a minute on an M3 MacBook Air
- The Mathematica evolution code for Hirai's method is approximately 10x to 100x slower than the equivalent Python code

## Context

This work extends the theoretical framework for transversity PDF evolution established by Hayashigaki (1997), Vogelsang (1997), and Hirai et al. (1998), whose Fortran implementation is no longer accessible. It provides an alternative to APFEL++ (Bertone, 2013; 2017), a C++ package that implements transversity evolution using a method similar to Hirai's. This is the first code package to incorporate both the Hirai t-step method and the Vogelsang Mellin moment method simultaneously, enabling users to choose between them based on accuracy requirements and computational cost. The code enables future comparisons with experimental data from semi-inclusive deep inelastic scattering and Drell-Yan processes, and can be extended to include the 3-loop anomalous dimension calculated by Blumlein (2021).
