# JuliaQCD: Portable lattice QCD package in Julia language

> We develop a new lattice gauge theory code set JuliaQCD using the Julia language. Julia is well-suited for integrating machine learning techniques and enables rapid prototyping and execution of algorithms for four dimensional QCD and other non-Abelian gauge theories. The code leverages LLVM for high-performance execution and supports MPI for parallel computations. Julia's multiple dispatch provides a flexible and intuitive framework for development. The code implements existing algorithms such as Hybrid Monte Carlo (HMC), many color and flavor, supports lattice fermions, smearing techniques, and full QCD simulations. It is designed to run efficiently across various platforms, from laptops to supercomputers, allowing for seamless scalability. The code set is currently available on GitHub https://github.com/JuliaQCD.

## Key claim

JuliaQCD is the first open-source lattice QCD code suite written entirely in Julia, aiming to minimize the combined time of algorithm implementation and execution by balancing rapid prototyping with performance comparable to Fortran 90, though ultimate high performance is not within its scope.

## Method

The authors developed a modular lattice gauge theory codebase in Julia, consisting of several packages (LatticeQCD.jl, Gaugefields.jl, LatticeDiracOperators.jl, QCDMeasurements.jl, Wilsonloop.jl) that implement standard algorithms including Hybrid Monte Carlo (HMC), Rational HMC (RHMC), heatbath, and overrelaxation for SU(N_c) gauge groups. The code supports Wilson, staggered, and domain-wall fermions, various smearing techniques (stout, gradient flow), and parallelization via MPI, with I/O in ILDG, JLD, and Bridge++ text formats.

## Key numbers

- Setup time: users can start working in less than 5 minutes without complex compilation steps
- Performance: competitive with traditional Fortran 90 codes
- Supported fermion flavors: Wilson (2 flavors), staggered (1-8 tastes), standard domain-wall (2 flavors)
- For staggered-fermion HMC, RHMC is automatically used for N_f other than 4 or 8; on Apple Silicon, N_f = 1-8 is available
- Supports general SU(N_c) gauge groups with optimized SU(2) and SU(3) implementations

## Context

This work extends the lattice QCD software landscape by providing a Julia-based alternative to established codes such as MILC, CPS, FermiQCD (Di Pierro, 2001), OpenQCD, Bridge++ (Akahoshi, 2021), Grid (Boyle, 2015), and SIMULATeQCD (HotQCD, 2023). It also complements Python-based tools like Lyncs-API (Bacchio, 2022), pyQCD (Spraggs, 2014), and the Grid Python Toolkit. JuliaQCD enables rapid prototyping of new algorithms, particularly those integrating machine learning techniques, and provides a portable framework that runs on any machine with Julia installed, from laptops to supercomputers.
