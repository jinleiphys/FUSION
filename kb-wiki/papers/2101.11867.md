# Simulating low-energy neutrino interactions with MARLEY

> Monte Carlo event generators are a critical tool for the interpretation of data obtained by neutrino experiments. Several modern event generators are available which are well-suited to the GeV energy scale used in studies of accelerator neutrinos. However, theoretical modeling differences make their immediate application to lower energies difficult. In this paper, I present a new event generator, MARLEY, which is designed to better address the simulation needs of the low-energy (tens of MeV and below) neutrino community. The code is written in C++14 with an optional interface to the popular ROOT data analysis framework. The current release of MARLEY (version 1.2.0) emphasizes simulations of the reaction $^{40}$Ar$(ν_e, e^{-})^{40}$K$^{*}$ but is extensible to other channels with suitable user input. This paper provides detailed documentation of MARLEY's implementation and usage, including guidance on how generated events may be analyzed and how MARLEY may be interfaced with external codes such as Geant4. Further information about MARLEY is available on the official website at http://www.marleygen.org.

## Key claim

MARLEY (Model of Argon Reaction Low Energy Yields) is a new Monte Carlo event generator designed specifically for simulating neutrino-nucleus scattering at energies of tens of MeV and below, where existing GeV-scale generators like GENIE, GiBUU, NEUT, and NuWro are inadequate due to their neglect of discrete nuclear level transitions and statistical de-excitation models. The current release (version 1.2.0) emphasizes simulations of the reaction ⁴⁰Ar(νₑ, e⁻)⁴⁰K* but is extensible to other channels, though realistic neutrino-nucleus scattering events may only be generated up to roughly 50 MeV due to the neglect of forbidden nuclear transitions above this energy.

## Method

MARLEY treats neutrino-nucleus scattering as a two-step process: first, a two-to-two scattering reaction is simulated using the allowed approximation differential cross section (long-wavelength and slow-nucleon limits) with precomputed nuclear matrix elements for Fermi and Gamow-Teller transitions; second, the excited residual nucleus de-excites independently using tabulated gamma-ray branching ratios for discrete levels at low excitation energies and the Hauser-Feshbach statistical model for compound nucleus decay at higher energies. The code is written in C++14 with an optional interface to the ROOT data analysis framework and uses the GNU Scientific Library for numerical routines.

## Key numbers

- The current release (version 1.2.0) emphasizes simulations of the reaction ⁴⁰Ar(νₑ, e⁻)⁴⁰K* but is extensible to other channels
- Realistic neutrino-nucleus scattering events may be generated up to roughly 50 MeV; above this energy, effects of forbidden nuclear transitions become increasingly important
- The code requires a minimum GCC version of 4.9.4 or a minimum Clang version of 3.5.2
- The maximum lab-frame kinetic energy of a recoiling ground-state nucleus in CEvNS is given by T_f^max = 2E_ν² / (2E_ν + M), where E_ν is the initial lab-frame neutrino energy and M is the nuclear target mass
- The differential cross section for CEvNS on a spin-zero target includes a nuclear recoil correction factor R(s) ≡ (s + M²)² / (4s²), which is usually approximated as 1 in the CEvNS literature

## Context

This work extends the approach of previous low-energy neutrino cross section calculations by Langanke (1996), Kolbe (1992, 2000, 2001, 2003), and others that treat neutrino-nucleus scattering as a two-step process with statistical de-excitation, but implements it as a standalone, documented event generator rather than a theoretical calculation. MARLEY fills a gap left by existing generators like GENIE, GiBUU, NEUT, and NuWro, which are optimized for GeV-scale accelerator neutrinos and lack native support for discrete gamma-ray transitions between low-lying nuclear levels that are critical for tens-of-MeV interactions. The code enables detailed simulations for experiments like DUNE (which will detect supernova and solar neutrinos via ⁴⁰Ar(νₑ, e⁻)⁴⁰K*) and HALO, and can be interfaced with external codes such as Geant4 for full detector simulations.
