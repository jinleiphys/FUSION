# Fast and accurate evaluation of Wigner 3j, 6j, and 9j symbols using prime factorisation and multi-word integer arithmetic

> We present an efficient implementation for the evaluation of Wigner 3j, 6j, and 9j symbols. These represent numerical transformation coefficients that are used in the quantum theory of angular momentum. They can be expressed as sums and square roots of ratios of integers. The integers can be very large due to factorials. We avoid numerical precision loss due to cancellation through the use of multi-word integer arithmetic for exact accumulation of all sums. A fixed relative accuracy is maintained as the limited number of floating-point operations in the final step only incur rounding errors in the least significant bits. Time spent to evaluate large multi-word integers is in turn reduced by using explicit prime factorisation of the ingoing factorials, thereby improving execution speed. Comparison with existing routines shows the efficiency of our approach and we therefore provide a computer code based on this work.

## Key claim

The paper presents a method for evaluating Wigner 3j, 6j, and 9j symbols that achieves a fixed relative accuracy bounded by six times the machine epsilon (6.66 × 10⁻¹⁶ for 64-bit floating point) while executing considerably faster than previous high-accuracy approaches, though the method is still slower than simple floating-point routines for small angular momenta.

## Method

The implementation expresses Wigner symbols in the form W_x = n√s / q, where n, s, and q are integers, using the Racah formula for 6j symbols and a rearranged binomial-based formula for 9j symbols. Each factorial is precomputed as a tuple of prime exponents using a sieve of Eratosthenes, and the least common denominator of the sum is found by taking the minimum exponent for each prime across all terms; the common factor is subtracted from each term's exponent tuple, the resulting pure numerators are converted to multi-word integers for exact accumulation, and the final floating-point value is computed from the combined integer result.

## Key numbers

- Maximum relative error bounded by 6ε = 6.66 × 10⁻¹⁶ for 64-bit floating-point format (IEEE 758)
- For the 6j symbol {2 2 2; 2 2 2}, the computed value is -27√1 / 630 = -0.0429
- For a 3j symbol with j=50,000, execution time is 112 seconds and memory usage is 19 GB
- For a 6j symbol with all j=50,000, execution time is 816 seconds and memory usage is 32 GB
- For a 9j symbol with all j=2,000, execution time is 358 seconds and memory usage is 109 MB
- For a 3j symbol with j=200, execution time is 0.31 ms and memory usage is 730 kB
- For a 6j symbol with all j=600, execution time is 6.1 ms and memory usage is 8.0 MB
- For a 9j symbol with all j=200, execution time is 0.15 s and memory usage is 1.6 MB
- Maximum factorial argument for 3j symbols is (3j+1)!, for 6j symbols is (4j+1)!, for 9j symbols is (5j+1)!
- Memory requirement for precomputed factorial tables at j=10,000: 0.78 GB for 3j, 1.35 GB for 6j, 2.06 GB for 9j

## Context

This work extends the approach of Wei (1999) who used prime factorisation and multi-word integer arithmetic with recursive calculation of binomial coefficients, but achieves significantly faster execution through precomputation of a table of prime-number factorisations. The method enables accurate evaluation of Wigner symbols for very large angular momenta (j up to 50,000 demonstrated) where floating-point routines such as GSL 1.16 suffer from accuracy losses that are five orders of magnitude larger already at j=15.
