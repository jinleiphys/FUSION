# dmscatter: A Fast Program for WIMP-Nucleus Scattering

> Recent work, using an effective field theory framework, has shown the number of possible couplings between nucleons and the dark-matter-candidate Weakly Interacting Massive Particles (WIMPs) is larger than previously thought. Inspired by an existing Mathematica script that computes the target response, we have developed a fast, modern Fortran code, including optional OpenMP parallelization, along with a user-friendly Python wrapper, to swiftly and efficiently explore many scenarios, with output aligned with practices of current dark matter searches. A library of most of the important target nuclides is included; users may also import their own nuclear structure data, in the form of reduced one-body density matrices. The main output is the differential event rate as a function of recoil energy, needed for modeling detector response rates, but intermediate results such as nuclear form factors can be readily accessed.

## Key claim

The authors present a fast Fortran code, dmscatter, that computes WIMP-nucleus scattering event rates using the effective field theory framework with up to 15 independent couplings, claiming extreme speedup over the existing Mathematica script dmformfactor while maintaining flexible nuclear structure input; the key qualifier is that the code assumes a harmonic oscillator single-particle basis and requires one-body density matrices from external many-body codes.

## Method

The code computes the differential event rate by integrating the product of nuclear and dark matter response functions weighted by the expected dark matter flux, using reduced one-body nuclear density matrices coupled to a multipole expansion of operators computed analytically in a harmonic oscillator basis. The nuclear structure input is provided as one-body density matrices from external configuration-interaction codes such as Bigstick, with a library of common target nuclides included. The Fortran engine includes optional OpenMP parallelization and is wrapped by a Python interface providing functions EventrateSpectra and NucFormFactor.

## Key numbers

- For 29Si with coupling c1^n: dmformfactor took 3,800 seconds; dmscatter serial took 0.5 seconds; parallel (4 threads) took 0.2 seconds (1000 recoil energies, WIMP mass 150 GeV)
- For 131Xe with coupling c1^n: dmformfactor took 20,000 seconds; dmscatter serial took 1.7 seconds; parallel took 0.5 seconds
- For 131Xe with all 15 couplings active: dmscatter serial took 75 seconds; parallel took 20 seconds
- A benchmark scanning 300 WIMP masses from 1 GeV to 300 GeV for 131Xe (1000 recoil energies each) took approximately 30 minutes CPU time with dmscatter, versus an estimated 70 days with dmformfactor
- The harmonic oscillator basis length b is set by default to b^2 = 41.467/(45A^{-1/3} - 25A^{-2/3}) fm^2 from the Blomqvist and Molinari prescription

## Context

This work extends the effective field theory framework for WIMP-nucleus scattering developed by Fitzpatrick et al. (2013) and Anand et al. (2014), who identified up to 15 independent nonrelativistic couplings and provided the Mathematica script dmformfactor. The code enables rapid exploration of the EFT parameter space including interference terms and sensitivity studies of nuclear physics uncertainties, complementing the DDCalc dark matter direct detection package (Bringmann et al. 2017, Athron et al. 2019) by offering more flexible nuclear structure input that allows users to import their own one-body density matrices.
