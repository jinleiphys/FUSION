# PairDiag: an exact diagonalization program for solving general pairing Hamiltonians

> We present a program for solving exactly the general pairing Hamiltonian based on diagonalization. The program generates the seniority-zero shell-model-like basis vectors via the `01' inversion algorithm. The Hamiltonian matrix is constructed in that seniority-zero space. The program evaluates all non-zero elements of the Hamiltonian matrix "on the fly" using the scattering operator and the search algorithm that act on the generated basis. The matrix is diagonalized by using the iterative Lanczos algorithm. The program thus developed, PairDiag, can calculate efficiently the ground-state eigenvalue and eigenvector of any pairing Hamiltonian. The program can be easily implemented to replace the BCS approximation in standard self-consistent mean-field calculations. The code is parallelized using OpenMP. For larger systems with dimension around 10$^{8-9}$, the calculation can be done within a day on standard desktop computers.

## Key claim

The paper presents PairDiag, an exact diagonalization program for general pairing Hamiltonians that can handle systems with dimension around 10^8-10^9 within a day on standard desktop computers, but this performance is conditional on the total number of orbits being less than 63 and the dimension being limited to that order for efficient diagonalization.

## Method

The program generates seniority-zero basis vectors using a `01' inversion algorithm that iteratively produces all binary representations of m pairs distributed in n orbits. The Hamiltonian matrix is constructed "on the fly" by evaluating non-zero elements through a scattering operator and search algorithm (binary or hash search) acting on the generated basis, without storing the full matrix. The sparse matrix is diagonalized using the iterative Lanczos algorithm with reorthogonalization and optional restart, and the code is parallelized using OpenMP.

## Key numbers

- For systems with dimension around 10^8-10^9, the calculation can be done within a day on standard desktop computers
- The total number of orbits allowed is less than 63 (using a 64-bit integer excluding the sign bit)
- The default Lanczos step size (Lanc_Limit) is 50, with recommended range between 10 and 50
- The default convergence control (Lanc_Error) is 1x10^-5
- The default dimension limit (Dimension_Limit) is 1x10^9
- For a space of C_26^13 with dimension about 10^7, approximately 50 iterations achieve convergence with good accuracy
- For a calculation with dimension N and iteration R, memory needed is about 8N(R+2)x10^-9 GB total (at least 41.6 GB for N=10^8 and R=50)
- In comparison with Lapack for C_16^8 (dimension 12870), the difference in ground-state eigenvalues between PairDiag and Lapack was at most 1x10^-10 for G values from -0.2 to -1.0
- In comparison with Richardson solver for systems up to C_38^10 (dimension 4.7x10^8) with G=-0.4, the difference between PairDiag and Richardson eigenvalues was at most 4x10^-7

## Context

This work extends exact diagonalization approaches for pairing Hamiltonians beyond previous limitations where dimension was limited to 10^6 on standard desktops (Liu 2013, Chang 2015). The program is designed to replace the BCS approximation in standard self-consistent mean-field calculations, addressing known problems with BCS such as particle-number violation and collapse around shell closures (Richardson 1966). The code can be combined with existing nuclear structure programs like EV8 (Bonche 2005) and Sky3D (Maruhn 2018), and the vector generation and search algorithms can be transplanted to large-scale shell model codes.
