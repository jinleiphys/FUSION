# Ultra fast, event-by-event heavy-ion simulations for next generation experiments

> We present a novel deep generative framework that uses probabilistic diffusion models for ultra fast, event-by-event simulations of heavy-ion collision output. This new framework is trained on UrQMD cascade data to generate a full collision event output containing 26 distinct hadron species. The output is represented as a point cloud, where each point is defined by a particle's momentum vector and its corresponding species information (ID). Our architecture integrates a normalizing flow-based condition generator that encodes global event features into a latent vector, and a diffusion model that synthesizes a point cloud of particles based on this condition. A detailed description of the model and an in-depth analysis of its performance is provided. The conditional point cloud diffusion model learns to generate realistic output particles of collision events which successfully reproduce the UrQMD distributions for multiplicity, momentum and rapidity of each hadron type. The flexible point cloud representation of the event output preserves full event-level granularity, enabling direct application to inverse problems and parameter estimation tasks while also making it easily adaptable for accelerating any event-by-event model calculation or detector simulation.

## Key claim

The paper presents a deep generative framework using probabilistic diffusion models that can produce complete, event-by-event heavy-ion collision outputs as point clouds, trained on UrQMD cascade data for a fixed beam energy (10 AGeV) and fixed impact parameter (b=1 fm), rather than claiming general applicability across all collision conditions without further conditioning.

## Method

The framework, called HEIDi, combines a PointNet-based encoder with a normalizing flow-based decoder to generate a latent condition vector encoding global event features, and a conditional diffusion model that synthesizes a point cloud of up to 1084 particles per event. Each particle in the point cloud is represented by its three momentum components (px, py, pz) and a one-hot encoded species identifier covering 26 distinct hadron types plus spectator nucleons and fake particles. The model was trained on 18000 Au-Au collision events at Elab = 10 AGeV with impact parameter b = 1 fm from the UrQMD cascade model.

## Key numbers

- Training dataset: 18000 Au-Au collision events at Elab = 10 AGeV with fixed impact parameter b = 1 fm
- Each event represented as a point cloud of 1084 particles (maximum multiplicity of training dataset), with each particle having 32 attributes (3 momentum components + 29 one-hot encoded species IDs)
- The model generates 26 distinct hadron species, with 16 most abundant species analyzed in detail; the remaining 10 species have mean multiplicities less than approximately 10^-2 per event
- HEIDi reproduces transverse momentum distributions across six orders of magnitude in probability for all hadron species
- Evaluation performed on 50000 HEIDi-generated events compared with 50000 UrQMD-generated events
- The model captures event-by-event multiplicity correlations between protons and pi+ (negative correlation), though not perfectly matching UrQMD

## Context

This work addresses the computational bottleneck in Bayesian inference for extracting the nuclear equation of state from heavy-ion collision data, where current surrogate models (Bernhard 2016, JETSCAPE 2020, Nijs 2020) are limited to a small number of computationally inexpensive observables. The approach extends prior generative modeling in heavy-ion physics (Sun 2024, Huang 2018) by generating complete event outputs rather than spectra or calorimeter responses, and builds on point cloud diffusion models (Luo 2021) adapted from computer vision. The framework enables direct application to inverse problems and parameter estimation tasks, and can be adapted to accelerate any event-by-event model calculation or detector simulation for next-generation experiments such as CBM at SIS-100, HADES at SIS-18, and MPD at NICA.
