# An implementation of hybrid parallel CUDA code for the hyperonic nuclear forces

> We present our recent effort to develop a GPGPU program to calculate 52 channels of the Nambu-Bethe-Salpeter (NBS) wave functions in order to study the baryon interactions, from nucleon-nucleon to $Ξ-Ξ$, from lattice QCD. We adopt CUDA programming to perform the multi-GPU execution on a hybrid parallel programming with MPI and OpenMP. Effective baryon block algorithm is briefly outlined, which calculates efficaciously a large number of NBS wave functions at a time, and three CUDA kernel programs are implemented to materialize the effective baryon block algorithm using GPUs on the single-program multiple-data (SPMD) programming model. In order to parallelize multiple GPUs, we take both two approaches by dividing the time dimension and by dividing the spatial dimensions. Performances are measured using HA-PACS supercomputer in University of Tsukuba, which includes NVIDIA M2090 and NVIDIA K20X GPUs. Strong scaling and weak scaling measured by using both M2090 and K20X GPUs are presented. We find distinct difference between the M2090 and the K20X in the sustained performance measurement of particular kernel executions which utilize the cudaStream objects.

## Key claim

The paper presents a hybrid parallel CUDA code for calculating 52 channels of Nambu-Bethe-Salpeter (NBS) wave functions for baryon interactions from lattice QCD, but the performance advantage of using GPUs is conditional: the cudaStream objects significantly improve performance only on the newer K20X GPUs (compute capability 3.5) and not on the older M2090 GPUs (compute capability 2.0), and no clear benefit or disadvantage is observed for performing the Fast Fourier Transform (FFT) on either CPUs or GPUs.

## Method

The authors implemented three CUDA kernel programs (NormalBaryonBlocks, EffectiveBaryonBlocks, and MultiplicationEffectiveBlocks) to compute the four-point correlation functions for 52 channels of baryon interactions, from nucleon-nucleon to Xi-Xi, using the effective baryon block algorithm. Multi-GPU parallelization was achieved through a hybrid MPI+OpenMP approach with single-program multiple-data (SPMD) programming, employing two strategies: dividing the time dimension and dividing the spatial dimensions. Performance was measured on the HA-PACS supercomputer using both NVIDIA M2090 (base cluster) and NVIDIA K20X (tightly coupled accelerators part) GPUs.

## Key numbers

- Peak double-precision performance: 665 GFlops for Tesla M2090, 1310 GFlops for Tesla K20Xm
- NormalBaryonBlocks kernel performance on single GPU: 98 GFlops (M2090) and 94 GFlops (K20X) without cudaStream; 98 GFlops (M2090) and 95 GFlops (K20X) with cudaStream
- EffectiveBaryonBlocks kernel performance on single GPU: 3.1 GFlops (M2090) and 0.55 GFlops (K20X)
- MultiplicationEffectiveBlocks kernel performance on single GPU: 8.2 GFlops (M2090) and 3.2 GFlops (K20X) without cudaStream; 8.0 GFlops (M2090) and 27 GFlops (K20X) with cudaStream (more than factor 8 improvement on K20X)
- All performance measurements used lattice size L^3 x T = 16^3 x 32 in double precision
- The effective baryon block algorithm reduced the number of iterations for the p-Lambda channel from 3456 (naive approach) to 370

## Context

This work extends the lattice QCD approach to nuclear forces pioneered by Ishii (2006) and Aoki (2009), which derives nucleon-nucleon potentials from NBS wave functions, and builds on the effective baryon block algorithm developed by Nemura (2009) that improved computational cost from O(L^6) to O(L^3 log L^3). The code is designed for a large-scale lattice QCD calculation in progress (Doi, Ishii, Sasaki 2015) to study baryon interactions from NN to Xi-Xi across 52 channels, and enables efficient multi-GPU execution on hybrid parallel architectures for first-principle nuclear physics calculations.
