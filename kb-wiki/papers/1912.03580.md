# EspressoDB: A scientific database for managing high-performance computing workflow

> Leadership computing facilities around the world support cutting-edge scientific research across a broad spectrum of disciplines including understanding climate change, combating opioid addiction, or simulating the decay of a neutron. While the increase in computational power has allowed scientists to better evaluate the underlying model, the size of these computational projects have grown to a point where a framework is desired to facilitate managing the workflow. A typical scientific computing workflow includes: Defining all input parameters for every step of the computation; Defining dependencies of computational tasks; Storing some of the output data; Post-processing these data files; Performing data analysis on output. EspressoDB is a programmatic object-relational data management framework implemented in Python and based on the Django web framework. EspressoDB was developed to streamline data management workflows, centralize and guarantee data integrity, while providing domain flexibility and ease of use. The framework provided by EspressoDB aims to support the ever increasing complexity of workflows of scientific computing at leadership computing facilities, with the goal of reducing the amount of human time required to manage the jobs, thus giving scientists more time to focus on science.

## Key claim

EspressoDB is a programmatic object-relational mapping (ORM) data management framework that aims to reduce the human time required to manage complex scientific computing workflows at leadership computing facilities, though it is designed to complement rather than replace existing workflow description tools like DVC, VisTrails, or Taverna.

## Method

The authors implemented EspressoDB in Python using the Django web framework, providing an object-relational mapping interface where database tables correspond to Python classes, rows to instances, and columns to attributes. The framework extends Django's ORM by allowing user-defined constraints not supported by SQL (such as unique constraints across related tables) and provides default configurations for documentation views, notification systems, project-specific Python interface guidelines, and admin pages. A specific application called LatteDB was created for lattice quantum chromodynamics (LQCD) calculations and is used by the CalLat Collaboration on the Summit supercomputer at the Oak Ridge Leadership Computing Facility.

## Key numbers

- Summit at OLCF is approximately 15 times faster than Titan when applied to LQCD applications such as those carried out by CalLat
- CalLat creates petabytes of temporary files written to the scratch file system, ultimately processed down to hundreds of terabytes saved for analysis
- A single LQCD calculation requires tens-of-thousands to millions of independent MPI tasks to be completed
- For each ensemble of gauge configurations, the computation requires O(1000) gauge configurations per ensemble and O(20) ensembles
- Each task generates multi-gigabyte data files, with most not saved permanently

## Context

This work extends the capabilities of existing ORM frameworks like SQLAlchemy and Django itself by lowering the barrier of access and providing features for uniqueness and consistency over multiple data dependencies. It contrasts with workflow description software such as DVC, VisTrails, or Taverna (described by Wolstencroft et al., 2013) by interacting with data itself rather than describing the workflow of computations. The framework enables more efficient management of the complex computational workflows required for LQCD calculations at near-exascale and exascale leadership computing facilities, with LatteDB currently supporting the CalLat Collaboration's computations on Summit through DOE INCITE Allocations.
