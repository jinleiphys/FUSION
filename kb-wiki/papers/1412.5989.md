# Accelerating Ab Initio Nuclear Physics Calculations with GPUs

> This paper describes some applications of GPU acceleration in ab initio nuclear structure calculations. Specifically, we discuss GPU acceleration of the software package MFDn, a parallel nuclear structure eigensolver. We modify the matrix construction stage to run partly on the GPU. On the Titan supercomputer at the Oak Ridge Leadership Computing Facility, this produces a speedup of approximately 2.2x - 2.7x for the matrix construction stage and 1.2x - 1.4x for the entire run.

## Key claim

GPU acceleration of the matrix construction stage in the MFDn nuclear structure eigensolver produces a speedup of approximately 2.2x to 2.7x for that stage and 1.2x to 1.4x for the entire run on the Titan supercomputer, but the overall speedup depends on the relative times of the matrix construction and diagonalization stages, which vary with the number of eigenvalues required and the convergence accuracy.

## Method

The authors modified the matrix construction stage of MFDn, a hybrid MPI/OpenMP parallel nuclear structure eigensolver, to run the basis transformation of three-body interaction matrix elements from the coupled-JT basis to the m-scheme basis on NVIDIA K20 Tesla GPUs using CUDA. Each GPU thread calculates one three-body element independently, and OpenMP threads buffer requests into chunks of approximately 20,000 elements before sending them to the GPU, while continuing CPU-based calculations for elements not yet processed by the GPU. The six-dimensional jagged index array for the coupled-JT interaction was flattened into a contiguous block on the CPU and copied to the GPU with a single allocation and copy to avoid excessive transfer overhead.

## Key numbers

- Speedup of 2.2x to 2.7x for the matrix construction stage on the Titan supercomputer (Cray XK7 with 16-core 2.2 GHz AMD Opteron 6274 processors and NVIDIA K20 Tesla GPUs)
- Overall run speedup of 1.2x to 1.4x for representative parameter choices
- Standalone basis transformation code achieved 4x to 10x speedup compared to a multithreaded CPU implementation running on eight cores (Oryspayev 2013)
- Buffers of approximately 20,000 elements were used for GPU requests, based on tests showing diminishing returns after that size on the Dirac supercomputer at NERSC
- In one representative case, the three-body interaction is 33 GB in the m-scheme basis but only 1 GB in the coupled-JT basis

## Context

This work extends GPU acceleration to the MFDn software package, which is used for ab initio no-core shell model calculations of nuclear structure (Navratil 2000, Vary 2009). The matrix construction stage typically consumes 25% to 45% of runtime in MFDn, and the authors target the computationally intensive basis transformation of three-body interactions from the coupled-JT basis to the m-scheme basis, a technique previously described by Maris (2013) and Roth (2011, 2013). The results enable larger-scale ab initio calculations on GPU-equipped supercomputers, though the authors note that GPU acceleration of the Lanczos diagonalization stage remains more difficult due to its memory-bound nature and irregular parallelism.
