# Accounting for backflow in hydrodynamic-simulation interfaces

> Methods for building a consistent interface between hydrodynamic and simulation modules is presented. These methods account for the backflow across the hydrodynamic/simulation hyper-surface. The algorithms are efficient, relatively straight-forward to implement, and account for conservation laws across the hyper-surface. The methods also account for the spurious interactions between particles in the backflow and other particles by following the subsequent impact of such particles. Since the number of altered trajectories grows exponentially in time, a cutoff is built into the procedure so that the effects of the backflow are ignored beyond a certain number of collisions

# Accounting for backflow in hydrodynamic-simulation interfaces

## Key claim

The paper presents a method to account for backflow across a hydrodynamic/simulation hyper-surface using negative-weight tracer particles, but the method is only correct to linear order in the backflow fraction and requires a cutoff on the number of collisions to prevent exponential growth of particle tracks and numerical noise.

## Method

The method generates particles at the hydrodynamic/cascade interface using the Cooper-Frye formula, where negative contributions (backflow) are represented by particles with weight -1. These tracer particles are evolved through the cascade alongside base particles (weight +1), with collision rules that conserve charge, energy, and momentum to linear order in the backflow. Tracer particles are allowed to interact with base particles but not with each other, and a cutoff \(N_{\rm max}\) limits the number of collisions a tracer particle can undergo to prevent exponential growth of the number of tracer trajectories.

## Key numbers

- Backflow at mid-rapidity is on the order of one half of one percent of total particles, based on prior work by Huovinen (2012) and Pratt (2008).
- Away from mid-rapidity, the error from neglecting backflow grows to several percent, as reported by Huovinen (2012).
- For a simulation of 100\(A\) GeV on 100\(A\) GeV Au+Au central collisions at RHIC, approximately 4.5 backflow particles per event were created when \(N_{\rm max}=0\), compared to over 2300 final-state base particles.
- With \(N_{\rm max}=0\), the backflow particles carried nearly 2.2 GeV of transverse energy (negative weight contribution to \(\Delta E_t\)).
- Setting \(N_{\rm max}=3\) simulated nearly 90% of the change in \(\Delta E_t\) while keeping the average number of tracer particles on the order of 100.
- Adding backflow corrections with \(N_{\rm max}=3\) slowed the B3D cascade code by 3% and increased final-state data file size by 5%.

## Context

This work extends prior hybrid hydrodynamic/cascade approaches that either ignored backflow entirely or attempted to erase particles crossing back into the hydrodynamic region, as in Pratt (2008). The method addresses the problem that the number of altered trajectories grows exponentially in time if all scatterings are followed, and provides a practical cutoff scheme. The approach enables more accurate modeling of the interface between hydrodynamics and hadronic cascades, particularly for regions away from mid-rapidity where backflow effects are larger, and is implemented in the B3D cascade code.
